## 日视图

### 布局规则

| 属性 | 规范 |
|------|------|
| 时间轴范围 | 08:00 - 20:00 |
| 时间刻度 | 每 30 分钟一格，每格高度 60px |
| 列定义 | 每列代表一位医生，列宽最小 180px，自适应平分 |
| 列头 | 显示医生头像(24px) + 姓名 + 今日预约数角标 |
| 当前时间线 | 红色水平虚线，标记当前时间，每分钟自动移动 |
| 非工作时段 | 灰色斜线背景，根据医生排班标记不可预约时段 |

### 交互行为

| 操作 | 行为 | 说明 |
|------|------|------|
| 点击空白时段 | 打开新建预约抽屉 | 自动预填: 点击的医生、日期、时段 |
| 点击预约卡片 | 打开预约详情抽屉 | 显示完整预约信息 |
| 右键预约卡片 | 弹出右键菜单 | 见下方"预约卡片右键菜单" |
| 拖拽预约卡片 (同列) | 改变预约时间 | 需二次确认弹窗 |
| 拖拽预约卡片 (跨列) | 更换医生+时间 | 需二次确认弹窗 |
| 鼠标悬停预约卡片 | 显示 Tooltip | Tooltip 内容: 患者电话、主诉、备注 |

## 周视图

| 条件 | 显示方式 |
|------|----------|
| 全部医生视图 | 卡片额外显示医生姓名缩写，卡片宽度较窄 |
| 单个医生视图 | 卡片完整显示患者姓名+诊疗类型+时段 |
| 同时段多个预约 | 横向并排排列，自动压缩宽度 |

## 预约卡片

### 卡片布局

```
┌─┬──────────────┐
│ │ 张三          │  ← 患者姓名 (加粗, 14px)
│色│ 09:00-09:30   │  ← 时段 (12px, #6B7280)
│条│ 洗牙          │  ← 诊疗类型标签 (Tag, 12px)
│ │ 初诊 👆        │  ← 可选标记 (初诊标签, 备注图标)
└─┴──────────────┘
```

### 卡片元素

| 元素 | 显示内容 | 样式说明 |
|------|----------|----------|
| 状态色条 | 左侧 4px 宽色条 | 根据状态着色 (见状态颜色表) |
| 患者姓名 | `patient.name` | 加粗, 14px, #111827, 超长截断加省略号 |
| 时段 | `start_time` - `end_time` | 12px, #6B7280, 格式: HH:MM-HH:MM |
| 诊疗类型 | `treatment_type` | Tag 标签展示, 12px |
| 初诊标记 | `is_first_visit` | 仅初诊时显示 "初诊" 小标签, 橙色 |
| 备注图标 | `notes` 是否有值 | 有备注时显示💬图标 |

## 新建/编辑预约表单 (抽屉, 480px, 右侧滑出)

### 表单字段

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 占位符 | 说明 |
|----------|--------|----------|------|----------|--------|------|
| 患者 | patient_id | SearchSelect | 是 | 必须选择已有患者或新建 | 搜索患者姓名/手机号 | 支持姓名/手机号模糊搜索，下拉列表显示患者姓名+手机号后4位；底部有 "+ 新建患者" 快捷入口 |
| 医生 | doctor_id | Select | 是 | 必选 | 请选择医生 | 选项格式: {医生姓名} · {专长}，按科室分组显示 |
| 诊室 | room_id | Select | 否 | | 请选择诊室 | 根据所选医生的排班自动推荐默认诊室 |
| 预约日期 | schedule_date | DatePicker | 是 | 不早于今天; 必须在医生排班日内 | 请选择日期 | 无排班的日期灰色不可选 |
| 开始时间 | start_time | TimePicker | 是 | 必须在排班时段内; 不能与已有预约冲突 | 请选择时间 | 30分钟粒度; 已占用时段灰色不可选 |
| 时长 | duration_minutes | Select | 是 | 必选 | 请选择时长 | 选项: 15分钟 / 30分钟 / 45分钟 / 60分钟 / 90分钟 / 120分钟; 默认30分钟 |
| 诊疗类型 | treatment_type | Select | 是 | 必选 | 请选择诊疗类型 | 选项: 检查(checkup) / 洗牙(cleaning) / 补牙(filling) / 拔牙(extraction) / 种植(implant) / 正畸(orthodontics) / 美白(whitening) / 根管治疗(root_canal) / 冠修复(crown) / 其他(other) |
| 诊疗详情 | treatment_type_detail | Textarea | 否 | 最大500字 | 请输入诊疗详情说明 | 仅选择 "其他" 时推荐填写 |
| 主诉 | chief_complaint | Textarea | 否 | 最大200字 | 请输入患者主诉 | |
| 是否初诊 | is_first_visit | Switch | 否 | | | 默认: 否; 开启时文案 "是", 关闭时文案 "否" |

| 备注 | notes | Textarea | 否 | 最大500字 | 请输入备注信息 | |

### 表单联动规则

| 触发字段 | 联动行为 |
|----------|----------|
| 选择医生 | 加载该医生可用排班日期，更新日期选择器可选范围 |
| 选择日期 | 加载该医生当天排班时段，更新时间选择器可选范围 |
| 选择时间+时长 | 自动计算结束时间并显示; 触发冲突检测 |
| 选择患者 | 如果是初诊患者自动开启 is_first_visit; 显示患者上次就诊简要信息 |

### 冲突检测

| 属性 | 规范 |
|------|------|
| 触发时机 | 选择医生+日期+时间+时长后自动触发 |
| 检测逻辑 | 调用 `/api/v1/appointments/conflict-check` 检测时段冲突 |
| 无冲突 | 显示绿色 ✓ 提示: "该时段可用" |
| 有冲突 | 显示红色 ✗ 警告: "与{患者姓名}的{诊疗类型}预约冲突 ({时段})"，禁止提交 |
| 诊室冲突 | 如选择了诊室，同时检测诊室占用: "该诊室在此时段已被占用" |

### 抽屉底部按钮

| 按钮 | 类型 | 说明 |
|------|------|------|
| 取消 | 默认按钮 | 关闭抽屉，如有未保存修改弹出确认弹窗 |
| 保存 | 主按钮 | 提交表单，保存后关闭抽屉并刷新日历 |
| 保存并继续 | 链接按钮 | 仅新建模式可见; 保存后清空表单，继续新建下一个预约 |

## 交互说明

### INT-01: 日历加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时根据当前视图模式计算日期范围 |
| 2 | 并行请求: 预约列表 + 医生列表 + 诊室列表 |
| 3 | 使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 数据返回后渲染预约卡片到对应时间格 |
| 5 | 自动滚动到当前时间附近位置 |

### INT-02: 筛选器交互

| 步骤 | 行为 |
|------|------|
| 1 | 切换医生/诊室筛选器时实时过滤日历显示 |
| 2 | 日视图: 筛选医生时隐藏/显示对应列 |
| 3 | 周视图/月视图: 筛选后仅统计匹配的预约 |
| 4 | 筛选状态持久化到 URL query 参数 |
| 5 | 清空筛选器 → 恢复显示全部 |

### INT-03: 视图切换

| 步骤 | 行为 |
|------|------|
| 1 | 切换视图时保持当前选中日期不变 |
| 2 | 重新计算日期范围并请求对应数据 |
| 3 | 切换动画: 淡入淡出 (fade), 时长 200ms |

### INT-04: 拖拽改期

| 步骤 | 行为 |
|------|------|
| 1 | 仅 scheduled/confirmed 状态的预约可拖拽 |
| 2 | 拖拽开始: 卡片变半透明 (opacity: 0.6), 目标位置显示蓝色虚线占位 |
| 3 | 放置后弹出确认弹窗: "确认将{患者姓名}的预约从{原时段}改到{新时段}？" |
| 4 | 确认 → 调用改期 API → 成功后刷新 → Toast 提示 "改期成功" |
| 5 | 取消 → 卡片回到原位 |
| 6 | 如目标时段有冲突 → 卡片回到原位 + Toast 提示 "该时段已有预约" |

### INT-05: 新建预约完成

| 步骤 | 行为 |
|------|------|
| 1 | 新建预约成功后关闭抽屉 |
| 2 | 日历自动刷新数据 |
| 3 | 自动滚动到新预约所在的时间位置 |
| 4 | 新预约卡片闪烁高亮 2 秒 (蓝色边框呼吸动画) |
| 5 | Toast 提示: "预约创建成功" |

### INT-06: 键盘快捷键

| 快捷键 | 行为 |
|--------|------|
| `N` | 打开新建预约抽屉 |
| `T` | 跳转到今天 |
| `←` / `→` | 前一天/后一天 (日视图), 前一周/后一周 (周视图) |
| `1` / `2` / `3` | 切换日视图/周视图/月视图 |
| `Esc` | 关闭当前抽屉/弹窗 |

## 状态机

### 预约状态流转

```
                    ┌──────────┐
        ┌──────────►│ cancelled │
        │           └──────────┘
        │
┌───────┴──┐    ┌───────────┐    ┌───────────┐    ┌─────────────┐    ┌───────────┐
│ scheduled ├───►│ confirmed  ├───►│ checked_in ├───►│ in_progress  ├───►│ completed  │
└───────┬──┘    └─────┬─────┘    └─────┬─────┘    └──────┬──────┘    └───────────┘
        │             │                │                  │
        │             │                │                  │
        ▼             ▼                ▼                  ▼
   ┌─────────┐  ┌─────────┐    ┌──────────┐        ┌──────────┐
   │ no_show  │  │ no_show  │    │ cancelled │        │ cancelled │
   └─────────┘  └─────────┘    └──────────┘        └──────────┘
```

### 状态流转规则

| 当前状态 | 可转为 | 操作权限 | 条件/说明 |
|----------|--------|----------|-----------|
| `scheduled` | `confirmed` | `appointment:confirm` | 医生/前台确认预约 |
| `scheduled` | `cancelled` | `appointment:cancel` | 需填取消原因 |
| `scheduled` | `no_show` | `appointment:no_show` | 预约时间过后仍未到 |
| `confirmed` | `checked_in` | `appointment:checkin` | 患者到店签到 |
| `confirmed` | `cancelled` | `appointment:cancel` | 需填取消原因 |
| `confirmed` | `no_show` | `appointment:no_show` | 预约时间过后仍未到 |
| `checked_in` | `in_progress` | `appointment:start` | 医生开始诊疗 |
| `checked_in` | `cancelled` | `appointment:cancel` | 签到后取消(罕见) |
| `in_progress` | `completed` | `appointment:complete` | 诊疗结束 |
| `in_progress` | `cancelled` | `appointment:cancel` | 诊中取消(罕见) |

| ≥ 1280px (桌面) | 完整日历视图，所有医生列并排 |
| 768-1279px (平板) | 日视图最多显示 3 列医生，可横向滚动; 工具栏折叠次要筛选器 |
| < 768px (手机) | 仅显示列表视图 (按时间排序的预约列表); 隐藏拖拽功能 |

## 空状态

| 场景 | 展示内容 |
|------|----------|
| 当天无预约 | 插画 + "今天还没有预约" + [新建预约] 按钮 |
| 筛选无结果 | 插画 + "未找到匹配的预约" + [清除筛选] 链接 |
| 医生未排班 | 对应列显示灰色背景 + "未排班" 文字 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取预约列表 | GET | `/api/v1/appointments` | `start_date`, `end_date`, `doctor_id[]`, `room_id[]`, `status[]` | 按日期范围和筛选条件获取预约 |
| 创建预约 | POST | `/api/v1/appointments` | 见表单字段 | 创建新预约 |
| 更新预约 | PUT | `/api/v1/appointments/:id` | 见表单字段 | 编辑已有预约 |
| 更新预约状态 | PATCH | `/api/v1/appointments/:id/status` | `status`, `reason` | 签到/开始/完成/取消状态变更 |
| 改期 | PATCH | `/api/v1/appointments/:id/reschedule` | `new_date`, `new_start_time`, `new_doctor_id`, `reason` | 预约改期 |
| 冲突检测 | POST | `/api/v1/appointments/conflict-check` | `doctor_id`, `date`, `start_time`, `duration_minutes`, `exclude_id` | 检测预约时段冲突 |
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active` | 获取在职医生列表 |
| 获取诊室列表 | GET | `/api/v1/rooms` | `status=active` | 获取可用诊室列表 |
| 获取医生排班 | GET | `/api/v1/schedules` | `doctor_id`, `start_date`, `end_date` | 获取医生排班信息 |
