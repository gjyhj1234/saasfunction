## 创建排班表单 (模态框, 560px)

### 表单字段

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 占位符 | 说明 |
|----------|--------|----------|------|----------|--------|------|
| 医生 | doctor_id | Text (只读) | 是 | - | - | 从左侧列表选中的医生, 显示头像+姓名+专长; 不可修改 |
| 排班模式 | schedule_mode | RadioGroup | 是 | 必选 | - | 单日排班 / 批量排班; 默认: 单日排班 |
| 日期 | schedule_date | DatePicker | 条件必填 | 不早于今天; 不能与已有排班冲突 | 请选择日期 | 单日排班模式下可见 |
| 日期范围 | date_range | DateRangePicker | 条件必填 | 起始不早于今天 | 请选择日期范围 | 批量排班模式下可见 |
| 开始时间 | start_time | TimePicker | 是 | 08:00-20:00 范围内; 30分钟粒度 | 请选择开始时间 | |
| 结束时间 | end_time | TimePicker | 是 | 必须晚于开始时间; 08:00-20:00 范围内; 30分钟粒度 | 请选择结束时间 | 开始与结束时间差至少 30 分钟 |
| 诊室 | room_id | Select | 否 | | 请选择诊室 | 可选分配诊室; 选项显示诊室名称+容纳设备数; 检测诊室同时段冲突 |
| 每个时段时长 | slot_duration_minutes | Select | 是 | 必选 | 请选择时段时长 | 选项: 15分钟 / 30分钟 / 45分钟 / 60分钟; 默认: 30分钟 |
| 最大预约数 | max_appointments | NumberInput | 是 | ≥1; ≤99; 整数 | 请输入最大预约数 | 该排班时段最多可接受的预约数量; 默认: 根据时段时长自动计算 |
| 重复规则 | repeat_rule | Select | 否 | | 请选择重复规则 | 选项: 不重复(none) / 每天(daily) / 每周(weekly) / 隔周(biweekly); 默认: 不重复; 仅单日排班模式可见 |
| 重复截止日期 | repeat_end_date | DatePicker | 条件必填 | 重复规则非 none 时必填; 必须晚于排班日期; 最远不超过 6 个月 | 请选择截止日期 | 仅当重复规则非 none 时可见 |
| 备注 | notes | Textarea | 否 | 最大 200 字 | 请输入备注 | |

### 排班模式说明

| 模式 | 说明 |
|------|------|
| 单日排班 | 为指定日期创建一条排班记录; 可设置重复规则 |
| 批量排班 | 为日期范围内的每天创建相同排班; 可勾选排除周末/节假日 |

### 批量排班额外选项

| 字段标签 | 字段名 | 组件类型 | 必填 | 说明 |
|----------|--------|----------|------|------|
| 排除周末 | exclude_weekends | Checkbox | 否 | 勾选后日期范围内的周六、周日不创建排班; 默认勾选 |
| 排除节假日 | exclude_holidays | Checkbox | 否 | 勾选后日期范围内的节假日不创建排班; 默认勾选 |
| 适用星期 | weekdays | CheckboxGroup | 否 | 周一/周二/周三/周四/周五/周六/周日; 默认: 周一到周五全选; 仅不勾选"排除周末"时可选周六周日 |

### 表单联动规则

| 触发字段 | 联动行为 |
|----------|----------|
| 选择开始时间+结束时间+时段时长 | 自动计算并显示可预约时段数和推荐最大预约数 |
| 选择诊室 | 自动检测诊室同时段是否被其他医生占用 |
| 选择重复规则 | none → 隐藏截止日期; 非 none → 显示截止日期(必填); 自动预览将创建的排班日期列表 |
| 切换排班模式 | 单日 ↔ 批量时清空日期/日期范围; 批量模式隐藏重复规则 |

### 冲突检测

| 检测类型 | 规则 | 提示内容 |
|----------|------|----------|
| 医生时段冲突 | 同一医生同日同时段已有排班 | 红色警告: "{医生}在{日期} {时段}已有排班" |
| 诊室冲突 | 同一诊室同日同时段已被分配 | 橙色警告: "{诊室}在{日期} {时段}已被{医生}使用" |
| 批量排班冲突 | 批量创建时部分日期有冲突 | 列表展示冲突日期，可选择跳过冲突日期继续创建 |

### 模态框底部按钮

| 按钮 | 类型 | 说明 |
|------|------|------|
| 取消 | 默认按钮 | 关闭模态框 |
| 预览 | 次按钮 | 仅批量/重复模式可见; 显示将要创建的所有排班日期列表 |
| 确认创建 | 主按钮 | 提交排班; 成功后关闭模态框，刷新日历; Toast 提示 "排班创建成功" |

## 交互说明

### INT-01: 医生选择与排班加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时默认选中医生列表第一位医生 |
| 2 | 右侧日历加载该医生的排班数据 |
| 3 | 切换选中医生时，右侧日历重新加载对应医生的排班 |
| 4 | 加载时使用骨架屏占位 |
| 5 | 医生列表实时显示每位医生的今日排班状态 |

### INT-02: 创建排班流程

| 步骤 | 行为 |
|------|------|
| 1 | 方式一: 点击 [创建排班] 按钮打开空白表单 |
| 2 | 方式二: 点击日历空白区域，预填医生+日期+时间后打开表单 |
| 3 | 填写表单，实时校验冲突 |
| 4 | 提交后日历自动刷新 |
| 5 | 新排班方块闪烁高亮 2 秒 |
| 6 | Toast 提示: "排班创建成功" / 批量: "成功创建 {N} 天排班" |

### INT-03: 拖拽调整排班

| 步骤 | 行为 |
|------|------|
| 1 | 仅可拖拽未来日期的排班 (过去的排班不可拖拽) |
| 2 | 拖拽开始: 方块变半透明 (opacity: 0.5), 目标位置显示蓝色虚线占位 |
| 3 | 放置后检测冲突 |
| 4 | 无冲突 → 弹出确认弹窗 |
| 5 | 有冲突 → 方块回到原位 + Toast 提示冲突信息 |
| 6 | 如排班时段内有已预约 → 确认弹窗额外提示: "此排班有 {N} 个已预约的预约，改动后预约不会自动调整" |
| 7 | 确认 → 更新排班 → Toast 提示 "排班调整成功" |

### INT-04: 节假日与排班联动

| 步骤 | 行为 |
|------|------|
| 1 | 新增停诊节假日后，日历对应日期标记红色背景 |
| 2 | 如该日期已有排班 → 弹出提示: "该日期已设为停诊日 ({节假日名称})，是否删除已有排班？" |
| 3 | 选择"是" → 删除排班; 选择"否" → 保留排班但日期标记为红色 (异常状态) |
| 4 | 停诊日不允许创建新排班 (创建时日期选择器灰色不可选) |

### INT-05: 诊室占用情况查看

| 步骤 | 行为 |
|------|------|
| 1 | 工具栏提供 "诊室视图" 切换按钮 |
| 2 | 切换后日历横轴变为诊室列 (而非日期列) |
| 3 | 每列显示一间诊室的占用情况 |
| 4 | 可快速发现诊室冲突和空闲时段 |

## 空状态

| 场景 | 展示内容 |
|------|----------|
| 该医生本周无排班 | 日历图标 + "本周暂无排班" + [创建排班] 按钮 |
| 无医生数据 | 人员图标 + "暂无医生信息" + [前往添加医生] 链接 |
| 节假日列表为空 | 日历图标 + "暂无节假日" + [添加节假日] 按钮 + [导入预设] 链接 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active`, `department_id` | 获取在职医生列表 |
| 获取排班列表 | GET | `/api/v1/schedules` | `doctor_id`, `start_date`, `end_date` | 获取指定医生指定日期范围的排班 |
| 创建排班 | POST | `/api/v1/schedules` | 见表单字段 | 创建单日排班 |
| 批量创建排班 | POST | `/api/v1/schedules/batch` | 见批量排班字段 | 批量创建多日排班 |
| 更新排班 | PUT | `/api/v1/schedules/:id` | 见编辑表单字段 | 更新排班信息 |
| 删除排班 | DELETE | `/api/v1/schedules/:id` | `delete_scope` (single/future/all) | 删除排班; 支持删除重复排班的不同范围 |
| 排班冲突检测 | POST | `/api/v1/schedules/conflict-check` | `doctor_id`, `date`, `start_time`, `end_time`, `room_id`, `exclude_id` | 检测排班时段冲突 |
| 获取诊室列表 | GET | `/api/v1/rooms` | `status=active` | 获取可用诊室 |
| 获取节假日列表 | GET | `/api/v1/holidays` | `year`, `country` | 获取节假日列表 |
| 创建节假日 | POST | `/api/v1/holidays` | 见节假日表单字段 | 创建节假日 |
| 更新节假日 | PUT | `/api/v1/holidays/:id` | 见节假日表单字段 | 更新节假日 |
| 删除节假日 | DELETE | `/api/v1/holidays/:id` | - | 删除节假日 |
| 导入预设节假日 | POST | `/api/v1/holidays/import-presets` | `country`, `year` | 一键导入指定国家预设节假日 |
| 获取诊室占用 | GET | `/api/v1/rooms/occupancy` | `date`, `start_time`, `end_time` | 查看诊室占用情况 |
