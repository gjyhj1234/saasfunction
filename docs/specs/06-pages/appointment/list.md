## 行操作

| 序号 | 按钮文本 | 操作类型 | 图标 | 权限 | 确认弹窗 | 可见条件 | 说明 |
|------|----------|----------|------|------|----------|----------|------|
| 1 | 查看 | 链接 | 👁 | `appointment:view` | 否 | 始终可见 | 打开预约详情抽屉 |
| 2 | 签到 | 操作 | ✅ | `appointment:checkin` | 否 | 仅 `scheduled` / `confirmed` 状态 | 状态变更为 `checked_in`; Toast 提示 "签到成功" |
| 3 | 改期 | 操作 | 📅 | `appointment:reschedule` | 是 | 仅 `scheduled` / `confirmed` 状态 | 打开改期弹窗 (同日历页改期弹窗) |
| 4 | 取消 | 危险操作 | ❌ | `appointment:cancel` | 是(需填原因) | 非 `completed` / `cancelled` 状态 | 打开取消确认弹窗; 红色文字 |
| 5 | 收费 | 链接 | 💰 | `billing:create` | 否 | 仅 `completed` 状态 | 跳转收费开单页 `/billing/charge?appointment_id={id}` |

### 行操作显示规则

| 条件 | 显示按钮 |
|------|----------|
| 超过 3 个可见按钮 | 前 2 个直接显示，其余收入 "更多" 下拉菜单 |
| 无任何可操作按钮 | 仅显示 "查看" |

## 批量操作

| 序号 | 按钮文本 | 操作类型 | 权限 | 确认弹窗 | 条件 | 说明 |
|------|----------|----------|------|----------|------|------|
| 1 | 批量确认 | 操作 | `appointment:confirm` | 是: "确认将 {N} 条预约标记为已确认？" | 至少选1条; 仅 `scheduled` 状态可批量确认 | 将选中的 `scheduled` 预约批量改为 `confirmed` |
| 2 | 批量取消 | 危险操作 | `appointment:cancel` | 是: "确认取消 {N} 条预约？此操作不可撤销" | 至少选1条; 非 `completed`/`cancelled` 状态 | 需选择统一的取消原因; 红色按钮 |
| 3 | 导出 | 导出 | `appointment:export` | 否 | 至少选1条 | 导出选中记录为 Excel; 如未选择则导出当前筛选结果 |

### 批量操作交互

| 行为 | 说明 |
|------|------|
| 选择行为 | 勾选复选框选择行; 表头复选框全选当前页 |
| 已选计数 | 表格上方显示 "已选 {N} 项" |
| 跨页选择 | 不支持跨页全选，仅支持当前页选择 |
| 状态不匹配 | 批量操作时自动跳过不满足条件的行，Toast 提示 "已跳过 {M} 条不符合条件的记录" |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时使用默认筛选条件 (今天的预约) 请求数据 |
| 2 | 并行请求: 预约列表 + 医生下拉 + 诊疗类型枚举 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-04: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 |
| 4 | 文件名格式: `预约列表_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 大数据量 (>1000条) 时显示导出进度条 |
| 6 | 导出完成 → Toast 提示 "导出成功" |

### INT-05: 状态快速切换

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [签到] / [开始诊疗] / [完成] 等操作按钮 |
| 2 | 无需确认弹窗，直接调用状态变更 API |
| 3 | 成功 → 表格行内状态 Badge 动画更新; Toast 提示 "{操作}成功" |
| 4 | 失败 → Toast 错误提示; 行数据不变 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取预约列表 | GET | `/api/v1/appointments` | `keyword`, `start_date`, `end_date`, `status[]`, `doctor_id`, `treatment_type`, `source`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取预约列表 |
| 获取预约详情 | GET | `/api/v1/appointments/:id` | - | 获取单个预约完整信息 (含患者信息、状态时间线) |
| 更新预约状态 | PATCH | `/api/v1/appointments/:id/status` | `status`, `reason` | 状态变更操作 |
| 批量更新状态 | PATCH | `/api/v1/appointments/batch-status` | `ids[]`, `status`, `reason` | 批量状态变更 |
| 导出预约 | POST | `/api/v1/appointments/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active` | 医生下拉选项 |
| 获取状态时间线 | GET | `/api/v1/appointments/:id/timeline` | - | 预约状态变更历史 |
