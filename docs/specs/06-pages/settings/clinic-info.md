## 表单字段

### 页面底部操作栏 (固定定位)

| 序号 | 按钮文本 | 类型 | 权限 | 说明 |
|------|----------|------|------|------|
| 1 | 取消 | 默认按钮 | - | 放弃未保存修改; 恢复为上次保存状态; 有修改时确认弹窗 "您有未保存的修改，确定放弃？" |
| 2 | 保存修改 | 主按钮 (Primary) | `settings:clinic_edit` | 保存所有区块修改; 表单无变更时按钮禁用 |

### 基本信息

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 机构名称 | `name` | Input | 是 | 2-100 字符 | 机构显示名称; 用于系统各处标题和报告 |
| 机构编码 | `code` | Input (只读) | 是 | 2-20 字符; 仅字母数字和横杠 | 创建后不可修改; 显示为只读灰色文本 |
| 所在国家 | `country` | Select | 是 | 仅限枚举值 | TH(泰国) / VN(越南) / PH(菲律宾) / ID(印度尼西亚) / MY(马来西亚) / SG(新加坡); ISO 3166-1 alpha-2 编码; 选项含国旗 emoji |
| 时区 | `timezone` | Select | 是 | 仅限 IANA 时区 | 按所在国家过滤可选时区; 如 Asia/Bangkok, Asia/Ho_Chi_Minh, Asia/Manila, Asia/Jakarta, Asia/Kuala_Lumpur, Asia/Singapore |
| 机构 Logo | `logo_url` | ImageUploader | 否 | PNG/JPG; 单张 ≤ 2MB; 建议 200×200px | 正方形 Logo; 用于系统左上角和报告头; 上传后实时预览; 支持删除恢复默认 Logo |
| 营业执照号 | `business_license_no` | Input | 否 | 最多 50 字符 | 商业登记编号; 用于打印发票和收据 |

### 联系方式

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 联系电话 | `contact_phone` | Input | 是 | 国际电话格式; 含国家区号; 如 +66xxxxxxxxx | 显示在预约通知和收据上; 按国家自动填充区号前缀 |
| 联系邮箱 | `contact_email` | Input | 是 | 有效邮箱格式; 最多 100 字符 | 用于系统通知发件人地址; 必须是有效可接收邮箱 |
| 地址 | `address` | Textarea | 是 | 最多 500 字符 | 机构详细地址; 显示在收据、预约确认等文件中; 支持多行 |

### 语言设置

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 默认语言 | `default_locale` | Select | 是 | 仅限枚举值 | en(English) / th(ไทย) / vi(Tiếng Việt) / id(Bahasa Indonesia) / ms(Bahasa Melayu); 新用户和访客使用此语言 |
| 支持的语言 | `supported_locales` | CheckboxGroup | 是 | 至少选 1 项; 必须包含默认语言 | 勾选后系统内可切换的语言列表; 默认语言自动勾选且不可取消 |

### 币种设置

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 默认币种 | `default_currency` | Select | 是 | 仅限枚举值 | THB(泰铢 ฿) / VND(越南盾 ₫) / PHP(菲律宾比索 ₱) / IDR(印尼盾 Rp) / MYR(林吉特 RM) / SGD(新加坡元 S$) / USD(美元 $); 新建账单默认币种 |
| 支持的币种 | `supported_currencies` | CheckboxGroup | 是 | 至少选 1 项; 必须包含默认币种 | 勾选后系统内可使用的币种列表; 默认币种自动勾选且不可取消; 币种选项含符号和名称 |

### 订阅信息 (只读)

### 支付渠道配置

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求机构信息详情 API |
| 2 | 并行请求: 机构详情 + 国家列表 + 时区列表 + 支付渠道配置 |
| 3 | 表单各区块使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 数据加载完成后填充所有表单字段 |
| 5 | 记录初始表单快照，用于变更检测 |
| 6 | 订阅信息区块根据到期时间显示警告: 到期前 30 天黄色提示 "订阅即将到期"，已过期红色提示 "订阅已过期" |

### INT-02: 国家切换联动

| 步骤 | 行为 |
|------|------|
| 1 | 切换所在国家后，时区下拉自动过滤为该国可用时区 |
| 2 | 如当前选中时区不在新国家可用列表中，自动选择该国默认时区 |
| 3 | 联系电话自动更新国家区号前缀: TH(+66) / VN(+84) / PH(+63) / ID(+62) / MY(+60) / SG(+65) |
| 4 | 支付渠道根据国家显示/隐藏区域性渠道: GCash 仅 PH 显示, OVO 仅 ID 显示 |

### INT-03: Logo 上传

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [上传] 或拖拽文件至上传区域 |
| 2 | 校验: PNG/JPG 格式; 文件 ≤ 2MB |
| 3 | 上传中显示进度条; 上传完成显示缩略图预览 |
| 4 | 点击 [删除] → 确认弹窗 "确认删除当前 Logo？删除后将恢复默认 Logo" |
| 5 | 格式不符 → Toast "仅支持 PNG/JPG 格式，且不超过 2MB" |

### INT-04: 语言设置交互

| 步骤 | 行为 |
|------|------|
| 1 | 默认语言变更时，自动确保该语言在支持语言列表中勾选 |
| 2 | 取消勾选某语言时，校验该语言是否为默认语言; 如果是则阻止取消并提示 "默认语言不可取消" |
| 3 | 取消勾选某语言时，提示 "取消该语言后，使用该语言的用户将切换至默认语言" |
| 4 | 至少保留 1 种支持语言 |

### INT-05: 币种设置交互

| 步骤 | 行为 |
|------|------|
| 1 | 默认币种变更时，自动确保该币种在支持币种列表中勾选 |
| 2 | 取消勾选某币种时，校验该币种是否为默认币种; 如果是则阻止取消并提示 "默认币种不可取消" |
| 3 | 取消勾选某币种时，校验是否有未结算的该币种账单; 如有则提示 "该币种有 {N} 笔未结算账单，取消后这些账单将保留原币种" |
| 4 | 至少保留 1 种支持币种 |

### INT-06: 支付渠道配置

| 步骤 | 行为 |
|------|------|
| 1 | 点击渠道行的 [配置] → 打开配置弹窗 (480px 宽度) |
| 2 | 弹窗内显示该渠道的启用开关和配置字段 |
| 3 | 开启渠道时，配置字段根据渠道类型动态显示必填项 |
| 4 | 密钥字段默认掩码显示; 点击可切换明文/掩码 |
| 5 | 点击 [保存] → 校验必填项 → 更新渠道配置 |
| 6 | 成功 → Toast "支付渠道配置已更新"; 关闭弹窗 |
| 7 | 关闭渠道时确认弹窗: "确认关闭 {渠道名称}？关闭后将无法使用该方式收款" |

### INT-07: 表单保存

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [保存修改] → 全表单验证 |
| 2 | 验证失败 → 定位到第一个错误字段; 字段下方显示红色错误提示; 区块标题显示红色错误计数标记 |
| 3 | 验证通过 → 调用更新机构信息 API |
| 4 | 保存成功 → Toast "机构信息已保存"; 更新表单初始快照 |
| 5 | 保存失败 → Toast 显示错误信息; 保持当前页面 |

### INT-08: 离开确认

| 步骤 | 行为 |
|------|------|
| 1 | 表单有修改且未保存时，导航离开触发确认弹窗 |
| 2 | 弹窗文案: "您有未保存的修改，确定离开？离开后修改将丢失" |
| 3 | [取消] 留在当前页; [确定离开] 放弃修改并跳转 |
| 4 | 适用场景: 点击 [取消]、点击侧边栏菜单、浏览器后退 |

### INT-09: 订阅信息显示

| 步骤 | 行为 |
|------|------|
| 1 | 计算距到期天数并显示在到期日期旁 |
| 2 | 剩余 > 30 天 → 正常显示 (绿色文本) |
| 3 | 剩余 1-30 天 → 黄色警告标签 "即将到期" |
| 4 | 已过期 → 红色标签 "已过期"; 页面顶部显示全局警告条 "您的订阅已过期，部分功能可能受限" |
| 5 | 显示 "如需升级套餐或续费，请联系客服" 提示文字和联系链接 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取机构信息 | GET | `/api/v1/tenant/current` | - | 获取当前租户的完整信息 |
