## 状态机/流转

```
┌──────────┐
│  active  │ (启用，正常使用)
└──┬───┬───┘
   │   │ 连续 5 次登录失败
   │   ▼
   │ ┌──────────┐
   │ │  locked  │ (锁定，无法登录)
   │ └────┬─────┘
   │      │ 管理员解锁
   │      ▼
   │ ┌──────────┐
   │ │  active  │ (解锁恢复)
   │ └──────────┘
   │
   │ 手动停用
   ▼
┌──────────┐
│ disabled │ (停用，无法登录)
└────┬─────┘
     │ 手动启用
     ▼
┌──────────┐
│  active  │ (重新启用)
└──────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `active` | `disabled` | 管理员点击 [停用] | 需确认; 不可停用自己; 停用后立即踢出登录会话 |
| `active` | `locked` | 连续 5 次密码错误 | 系统自动锁定; 锁定 30 分钟后自动解锁或管理员手动解锁 |
| `disabled` | `active` | 管理员点击 [启用] | 无需确认; 直接启用 |
| `locked` | `active` | 管理员点击 [解锁] 或 30 分钟自动解锁 | 解锁后重置登录失败计数 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求员工列表 (默认筛选: status=active) |
| 2 | 并行请求: 员工列表 + 角色枚举 + 科室列表 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建员工

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建员工] → 打开新建员工抽屉 (从右侧滑入, 640px) |
| 2 | 抽屉标题: "新建员工" |
| 3 | 默认显示: 账号信息、联系方式、角色分配、科室区块 |
| 4 | 用户名输入时实时查重 (300ms 防抖) |
| 5 | 密码强度实时提示: 弱(红色) / 中(黄色) / 强(绿色) |
| 6 | 勾选"医生"角色 → 动态展开医生信息区块 (动画过渡) |
| 7 | 取消勾选"医生"角色 → 收起医生信息区块; 清空已填数据需确认 |
| 8 | 点击 [保存] → 全表单验证 → 调用创建员工 API |
| 9 | 成功 → Toast "员工创建成功"; 关闭抽屉; 刷新列表 |
| 10 | 失败 → Toast 错误提示; 抽屉保持打开 |

### INT-04: 编辑员工

| 步骤 | 行为 |
|------|------|
| 1 | 点击姓名链接 / [编辑] → 打开编辑员工抽屉，预填当前数据 |
| 2 | 用户名字段只读不可修改 |
| 3 | 不显示密码和确认密码字段 (密码通过重置功能修改) |
| 4 | 如员工有医生角色，医生信息区块自动展开并预填数据 |
| 5 | 执业证号如已过期，字段旁显示红色警告 "证书已过期" |
| 6 | 点击 [保存] → 调用更新员工 API |
| 7 | 成功 → Toast "保存成功"; 关闭抽屉; 刷新列表 |

### INT-05: 重置密码

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [重置密码] → 打开重置密码弹窗 (400px) |
| 2 | 输入新密码和确认密码 |
| 3 | 密码规则: 8-32 字符; 含大小写字母和数字 |
| 4 | 点击 [确认重置] → 调用重置密码 API |
| 5 | 成功 → Toast "密码已重置"; 关闭弹窗; 该员工所有活跃会话失效 |
| 6 | 不可为自己重置密码 (自己的密码通过个人设置修改) |

### INT-06: 停用员工

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [停用] → 弹出确认弹窗 |
| 2 | 校验: 不可停用自己的账号 → Toast "不能停用自己的账号" |
| 3 | 校验: 员工是否有未完成的预约 → 提示 "该员工有 {N} 个未完成预约，停用后这些预约需要重新分配" |
| 4 | 确认后调用停用 API |
| 5 | 成功 → Toast "员工已停用"; 行内状态实时更新; 被停用员工立即被踢出登录 |

### INT-07: 解锁员工

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [解锁] → 无需确认，直接调用解锁 API |
| 2 | 成功 → Toast "账号已解锁"; 行内状态更新为 active |
| 3 | 重置该员工的登录失败计数为 0 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含: 姓名、用户名、角色、科室、电话、邮箱、状态、创建时间 |
| 4 | 不导出密码等敏感信息 |
| 5 | 文件名格式: `员工列表_{YYYYMMDD_HHmmss}.xlsx` |
| 6 | 导出完成 → Toast "导出成功" |

### INT-09: 医生信息联动

| 步骤 | 行为 |
|------|------|
| 1 | 角色 CheckboxGroup 中勾选"医生"角色时，表单底部动态展开"医生信息"区块 |
| 2 | 展开时带滑入动画 (300ms ease-in-out) |
| 3 | 医生信息中的执业证号和证书有效期变为必填 |
| 4 | 取消勾选"医生"时，如医生信息区块有已填数据，弹出确认: "取消医生角色将清空已填写的医生信息，是否继续？" |
| 5 | 确认 → 清空医生信息并收起区块; 取消 → 保持不变 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取员工列表 | GET | `/api/v1/users` | `keyword`, `role`, `department_id`, `status`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取员工列表 |
| 获取员工详情 | GET | `/api/v1/users/:id` | - | 含角色、科室、医生信息 |
| 创建员工 | POST | `/api/v1/users` | 见下方请求体 | 新建员工 |
| 更新员工 | PUT | `/api/v1/users/:id` | 见下方请求体 (不含 password) | 编辑员工信息 |
| 停用员工 | PATCH | `/api/v1/users/:id/disable` | - | 停用员工账号 |
| 启用员工 | PATCH | `/api/v1/users/:id/enable` | - | 启用员工账号 |
| 解锁员工 | PATCH | `/api/v1/users/:id/unlock` | - | 解锁锁定账号 |
| 重置密码 | PATCH | `/api/v1/users/:id/reset-password` | `new_password` | 管理员重置员工密码 |
| 校验用户名 | GET | `/api/v1/users/check-username` | `username` | 用户名唯一性校验 |
| 上传头像 | POST | `/api/v1/upload/image` | `file` (multipart), `type=user_avatar` | 上传头像至 CDN |
| 获取角色列表 | GET | `/api/v1/roles` | - | 可分配的角色列表 |
| 获取科室列表 | GET | `/api/v1/departments` | `status=active` | 可选科室列表 |
| 导出员工 | POST | `/api/v1/users/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
