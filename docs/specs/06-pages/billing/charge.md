## 状态机/流转

```
┌──────────┐
│ pending  │ (待支付，收费单已创建)
└────┬─────┘
     │ 全额支付 ──────────────────────→ ┌──────────┐
     │                                  │  paid    │ (已结清)
     │ 部分支付                          └──────────┘
     ▼                                       ▲
┌──────────┐                                 │
│ partial  │ ── 补齐余额 ───────────────────→─┘
└────┬─────┘
     │ 超过账期
     ▼
┌──────────┐       ┌──────────────┐
│ overdue  │──────→│ written_off  │ (坏账核销)
└────┬─────┘ 核销  └──────────────┘
     │
     │ 退款申请 (任何已支付状态)
     ▼
┌──────────┐
│ refunded │ (已退款)
└──────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `pending` | `paid` | 全额支付成功 | paid_amount = total_amount |
| `pending` | `partial` | 部分支付 | 0 < paid_amount < total_amount |
| `pending` | `overdue` | 超过账期未支付 | 系统定时任务自动标记 |
| `partial` | `paid` | 补齐余额 | outstanding_amount = 0 |
| `partial` | `overdue` | 超过账期未结清 | 系统定时任务自动标记 |
| `overdue` | `paid` | 全额还款 | |
| `overdue` | `written_off` | 坏账核销审批通过 | 需 `billing:refund` 权限 |
| `paid` | `refunded` | 退款申请通过 | 需 `billing:refund` 权限; 全额或部分退款 |
| `partial` | `refunded` | 退款申请通过 | 退还已付部分 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时检查 URL 参数: `patient_id`, `appointment_id`, `treatment_plan_id` |
| 2 | 如有 `patient_id` → 自动加载患者信息卡片 + 当天预约列表 + 进行中方案列表 |
| 3 | 如有 `appointment_id` → 自动关联预约并预填医生 |
| 4 | 如有 `treatment_plan_id` → 自动导入方案当前步骤费用项目到明细表 |
| 5 | 并行请求: 费用项目分类树 + 医生列表 + 币种配置 + 税率配置 |
| 6 | 表单区域使用骨架屏 (Skeleton) 作为加载占位 |

### INT-02: 患者选择

| 步骤 | 行为 |
|------|------|
| 1 | 在搜索框输入关键词，300ms 防抖后弹出下拉候选列表 |
| 2 | 候选列表显示: 姓名、手机号、患者编号、会员等级 |
| 3 | 选中患者 → 加载并显示患者信息卡片 (姓名/会员/余额/积分) |
| 4 | 自动加载该患者当天预约 + 进行中治疗方案 |
| 5 | 如患者有会员折扣 → 自动应用到后续添加的费用项目折扣率 |
| 6 | 点击 [快速建档] → 打开新建患者抽屉，创建完成后自动选中 |

### INT-03: 添加费用项目

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [+ 添加费用项目] 打开费用项目选择弹窗 |
| 2 | 弹窗左侧显示分类树，右侧显示项目列表 (含名称/编码/单价) |
| 3 | 支持搜索、分类筛选 |
| 4 | 点选项目 → 追加到费用明细表; 可连续多选 |
| 5 | 添加后自动填入项目名称/编码/单位/默认单价 |
| 6 | 如患者有会员折扣 → 折扣率字段自动预填会员折扣百分比 |
| 7 | 关闭弹窗后焦点回到费用明细表 |

### INT-04: 金额实时计算

| 步骤 | 行为 |
|------|------|
| 1 | 任何费用项目的数量/单价/折扣率变更时，实时重算行小计 |
| 2 | 行小计 = 数量 × 单价 × (1 - 折扣率/100) |
| 3 | 小计 = 所有行小计之和 |
| 4 | 整单折扣金额根据 discount_type 计算 |
| 5 | 税费 = (小计 - 折扣金额) × 税率 |
| 6 | 应收合计 = 小计 - 折扣金额 + 税费 |
| 7 | 实付金额 = 应收合计 - 积分抵扣 - 储值抵扣 - 优惠券 |
| 8 | 所有金额变更均有数字滚动动画过渡效果 |

### INT-05: 支付提交

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [提交收费] → 前端校验: 患者已选、至少 1 个费用项目、支付方式已选 |
| 2 | 校验通过 → 弹出确认弹窗显示费用汇总和支付方式 |
| 3 | 点击 [确认] → 调用创建账单 API |
| 4 | 成功 → Toast "收费成功"; 如全额支付跳转账单详情页; 如部分支付生成应收记录 |
| 5 | 如开启分期 → 同时创建分期计划 |
| 6 | 失败 → Toast 错误提示; 表单数据保持不变 |

### INT-06: 组合支付

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [+ 添加支付方式] 增加一行支付方式 |
| 2 | 每行可选不同的支付方式和金额 |
| 3 | 实时校验所有支付金额之和 = 实付金额 |
| 4 | 不等时显示差额提示: "还差 ฿{差额}" 或 "多付 ฿{差额}" |
| 5 | 储值余额支付自动校验余额充足性 |

### INT-07: 会员优惠自动计算

| 步骤 | 行为 |
|------|------|
| 1 | 选中会员患者后，自动识别会员等级 |
| 2 | 显示可用积分和储值余额 |
| 3 | 新增费用项目时自动填入会员折扣率 |
| 4 | 积分抵扣显示换算关系: {N}积分 = ฿{M} |
| 5 | 提交后自动计算本次消费获得的积分 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 搜索患者 | GET | `/api/v1/patients/search` | `keyword`, `page`, `page_size` | 患者搜索下拉; 返回姓名/手机号/编号/会员等级 |
| 获取患者详情 | GET | `/api/v1/patients/:id` | - | 获取患者完整信息 (含会员/余额/积分) |
| 获取患者预约 | GET | `/api/v1/patients/:id/appointments` | `date`, `status=completed` | 获取患者指定日期的预约 |
| 获取患者治疗方案 | GET | `/api/v1/patients/:id/treatment-plans` | `status=in_progress` | 获取患者进行中的治疗方案 |
| 获取费用项目列表 | GET | `/api/v1/fee-items` | `keyword`, `category`, `is_package`, `price_min`, `price_max`, `status=active`, `page`, `page_size` | 费用项目选择弹窗 |
| 获取费用项目分类树 | GET | `/api/v1/fee-items/categories` | - | 分类树结构数据 |
| 创建账单 | POST | `/api/v1/billing-records` | `patient_id`, `appointment_id`, `treatment_plan_id`, `doctor_id`, `currency`, `items[]`, `discount_type`, `discount_value`, `coupon_id`, `member_points_used`, `stored_value_used`, `is_installment`, `notes` | 创建收费记录 |
| 创建支付 | POST | `/api/v1/billing-records/:id/payments` | `amount`, `currency`, `payment_method`, `transaction_id` | 创建支付记录; 支持多笔 (组合支付) |
| 创建分期计划 | POST | `/api/v1/installment-plans` | `patient_id`, `billing_record_id`, `total_amount`, `currency`, `installment_count`, `frequency`, `start_date`, `first_payment` | 创建分期还款计划 |
| 验证优惠券 | POST | `/api/v1/coupons/validate` | `coupon_id`, `patient_id`, `amount` | 验证优惠券有效性和抵扣金额 |
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active` | 医生下拉选项 |
| 获取税率配置 | GET | `/api/v1/settings/tax-rate` | - | 当前诊所税率配置 |
| 获取币种配置 | GET | `/api/v1/settings/currencies` | - | 支持的币种列表和汇率 |
