## 状态机/流转

```
┌──────────┐
│ pending  │ (待收款，账单已创建)
└────┬─────┘
     │ 部分收款                    ┌──────────────┐
     ├────────────────────────────→│ partial_paid │
     │                             └──────┬───────┘
     │ 全额收款                           │ 补齐余额
     ├──────────────────┬─────────────────┘
     │                  ▼
     │            ┌──────────┐
     ├───────────→│   paid   │ (已结清)
     │            └──────────┘
     │ 超过账期
     ▼
┌──────────┐    全额/部分收款    ┌──────────────┐
│ overdue  │───────────────────→│ paid /       │
│          │                    │ partial_paid │
└────┬─────┘                    └──────────────┘
     │ 坏账核销 (逾期>90天)
     ▼
┌──────────────┐
│ written_off  │ (已核销)
└──────────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `pending` | `paid` | 全额收款 | outstanding_amount = 0 |
| `pending` | `partial_paid` | 部分收款 | 0 < paid < original |
| `pending` | `overdue` | 超过 due_date | 系统定时任务每日 00:00 扫描 |
| `partial_paid` | `paid` | 补齐余额 | outstanding_amount = 0 |
| `partial_paid` | `overdue` | 超过 due_date 且有余额 | 系统定时任务自动标记 |
| `overdue` | `paid` | 全额还款 | |
| `overdue` | `partial_paid` | 部分还款但未清 | |
| `overdue` | `written_off` | 坏账核销 | 需 `receivable:write_off` 权限; 逾期 >90天 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求应收列表 (默认筛选: pending + partial_paid + overdue, 按欠款金额降序) |
| 2 | 并行请求: 应收列表 + 汇总统计 + 币种配置 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 统计卡片使用数字滚动动画加载 |
| 5 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-04: 收款操作

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [收款] → 打开收款弹窗，预填欠款金额 |
| 2 | 选择支付方式，填写收款金额 (可部分收款) |
| 3 | 点击 [确认收款] → 调用创建支付 API |
| 4 | 成功 → Toast "收款成功"; 行内状态实时更新; 统计卡片刷新 |
| 5 | 如全额收款 → 状态变为 `paid`; 如部分 → 状态变为 `partial_paid` |
| 6 | 失败 → Toast 错误提示; 弹窗保持打开 |

### INT-05: 催款发送

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [催款] → 弹出催款设置弹窗 |
| 2 | 选择催款渠道 (WhatsApp/SMS/邮件)，选择催款模板 |
| 3 | 预览催款内容 → 确认发送 |
| 4 | 成功 → Toast "催款通知已发送"; reminder_count +1; last_reminder_sent_at 更新 |
| 5 | 失败 → Toast 错误提示; 显示失败原因 (如手机号无效/WhatsApp 未注册) |

### INT-06: 创建分期

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [创建分期] → 打开分期计划弹窗 |
| 2 | 填写分期期数、频率、起始日期 |
| 3 | 系统自动预览分期计划: 每期金额和到期日 |
| 4 | 确认 → 创建分期计划; 应收记录关联分期 |
| 5 | Toast "分期计划已创建，共 {N} 期" |

### INT-07: 坏账核销

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [核销] → 弹出核销确认弹窗 (红色警告) |
| 2 | 填写核销原因 (必填) |
| 3 | 确认 → 调用核销 API |
| 4 | 成功 → 状态变更为 `written_off`; Toast "已核销 {金额}" |
| 5 | 行样式变为灰色删除线，排序降低 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 + 账龄分析 |
| 4 | 文件名格式: `应收账款_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 大数据量 (>1000条) 时显示导出进度条 |
| 6 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取应收列表 | GET | `/api/v1/receivables` | `keyword`, `status[]`, `overdue_range`, `currency`, `due_date_start`, `due_date_end`, `amount_min`, `amount_max`, `is_installment`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取应收账款列表 |
| 获取应收汇总 | GET | `/api/v1/receivables/summary` | 同列表筛选参数 (不含分页) | 统计卡片数据: 应收总额/逾期金额/逾期笔数/本月收回 |
| 获取应收详情 | GET | `/api/v1/receivables/:id` | - | 含患者信息/账单信息/收款历史/分期计划 |
| 收款 | POST | `/api/v1/receivables/:id/collect` | `amount`, `payment_method`, `transaction_id`, `payment_time`, `notes` | 录入收款; 自动更新 outstanding_amount 和状态 |
| 发送催款 | POST | `/api/v1/receivables/:id/remind` | `channel[]`, `template_id`, `message` | 发送催款通知 |
| 批量催款 | POST | `/api/v1/receivables/batch-remind` | `ids[]`, `channel[]`, `template_id`, `message` | 批量发送催款通知 |
| 坏账核销 | POST | `/api/v1/receivables/:id/write-off` | `reason`, `approval_notes` | 坏账核销; 需审批权限 |
| 批量核销 | POST | `/api/v1/receivables/batch-write-off` | `ids[]`, `reason`, `approval_notes` | 批量核销 |
| 创建分期计划 | POST | `/api/v1/installment-plans` | `patient_id`, `billing_record_id`, `total_amount`, `currency`, `installment_count`, `frequency`, `start_date`, `first_payment` | 创建分期还款计划 |
| 获取分期详情 | GET | `/api/v1/installment-plans/:id` | - | 分期计划详情 (含每期明细) |
| 导出应收 | POST | `/api/v1/receivables/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 (含账龄分析) |
| 获取催款模板 | GET | `/api/v1/reminder-templates` | `type=receivable` | 催款消息模板列表 |
| 获取币种配置 | GET | `/api/v1/settings/currencies` | - | 支持的币种列表和汇率 |
