## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求随访计划列表 (默认按下次任务日期升序, 进行中优先) |
| 2 | 并行请求: 计划列表 + 医生下拉 + 计划类型枚举 |
| 3 | 列表使用骨架屏作为加载占位 |
| 4 | 检查 URL query 参数: `patient_id` / `status` / `plan_type` |
| 5 | 逾期任务行背景色为浅红色 (#FEF2F2) 高亮提示 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索范围: 患者姓名、计划名称、医生姓名 |
| 3 | 搜索时显示 loading 状态 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建随访计划

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建随访计划] 打开新建抽屉 (600px) |
| 2 | 选择患者 → 自动加载该患者的预约和治疗记录 |
| 3 | 选择计划类型 → 自动生成计划名称 |
| 4 | 可选: 选择模板 → 预览模板任务列表 → 确认后自动填充任务 |
| 5 | 手动模式: 逐条添加任务 (日期 + 描述 + 提前提醒天数) |
| 6 | 选择提醒渠道 (根据患者联系方式自动推荐: 有 WhatsApp → 默认 WhatsApp) |
| 7 | 点击 [确认创建] → 创建计划; Toast "随访计划已创建" |
| 8 | 成功后关闭抽屉，列表自动刷新 |

### INT-04: 查看详情

| 步骤 | 行为 |
|------|------|
| 1 | 点击计划名称或 [查看详情] 打开详情抽屉 (600px) |
| 2 | 显示计划基本信息 + 任务时间线 + 提醒记录 |
| 3 | 任务时间线: 已完成(绿色圆点✅) / 待执行(空心圆点⏳) / 失约(红色圆点❌) / 已取消(灰色圆点) |
| 4 | 待执行任务显示 [发送提醒] 和 [创建预约] 快捷操作 |
| 5 | 已完成任务显示患者反馈和备注 |

### INT-05: 发送提醒

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [发送提醒] 弹出确认弹窗，显示渠道和患者信息 |
| 2 | 确认后调用发送提醒接口 |
| 3 | 成功 → Toast "提醒已发送至 {channel}"; 更新提醒状态 |
| 4 | 失败 → Toast (error) "提醒发送失败: {error_message}"; 可重试 |

### INT-06: 创建预约

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [创建预约] 跳转至预约创建页 `/appointments/create` |
| 2 | URL 参数携带: `patient_id`, `doctor_id`, `follow_up_plan_id`, `task_id` |
| 3 | 预约创建页自动预填患者和医生信息 |
| 4 | 预约创建成功后, 对应任务的 `appointment_id` 自动关联 |

### INT-07: 取消计划

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [取消计划] 弹出确认弹窗 |
| 2 | 确认后将计划状态变更为 cancelled |
| 3 | 所有 pending 状态的任务同时变更为 cancelled |
| 4 | 已完成的任务保持不变 |
| 5 | Toast "随访计划已取消"; 列表刷新 |

|------|----------|
| 无随访计划 | 日历插画 + "暂无随访计划" + [新建随访计划] 按钮 |
| 搜索无结果 | 搜索插画 + "未找到匹配的随访计划" + "请尝试其他搜索词或调整筛选条件" |
| 筛选无结果 | 筛选插画 + "没有符合条件的随访计划" + [清除筛选] 链接按钮 |
| 详情无任务 | 空列表插画 + "该计划暂无任务" + [添加任务] 按钮 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取随访计划列表 | GET | `/api/v1/follow-up/plans` | `keyword`, `plan_type`, `status`, `doctor_id`, `reminder_channel`, `created_at_start`, `created_at_end`, `next_task_start`, `next_task_end`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取随访计划列表 |
| 获取随访计划详情 | GET | `/api/v1/follow-up/plans/:id` | - | 含完整信息 + 任务列表 + 提醒记录 |
| 新建随访计划 | POST | `/api/v1/follow-up/plans` | `patient_id`, `doctor_id`, `appointment_id`, `treatment_record_id`, `plan_type`, `plan_name`, `description`, `reminder_channel`, `tasks[]` | 创建随访计划及其任务 |
| 更新随访计划 | PUT | `/api/v1/follow-up/plans/:id` | `plan_name`, `description`, `reminder_channel`, `tasks[]` | 更新计划信息和任务 |
| 取消随访计划 | PUT | `/api/v1/follow-up/plans/:id/cancel` | - | 取消计划及未完成任务 |
| 批量取消计划 | PUT | `/api/v1/follow-up/plans/batch-cancel` | `ids[]` | 批量取消 |
| 发送提醒 | POST | `/api/v1/follow-up/plans/:id/send-reminder` | `task_id` | 向患者发送随访提醒 |
| 批量发送提醒 | POST | `/api/v1/follow-up/plans/batch-send-reminder` | `ids[]` | 批量发送提醒 |
| 更新任务结果 | PUT | `/api/v1/follow-up/tasks/:id/result` | `result`, `actual_date`, `patient_feedback`, `notes` | 更新任务执行结果 |
| 获取模板列表 | GET | `/api/v1/follow-up/templates` | `treatment_type`, `status=active` | 获取可用的随访模板 |
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active` | 医生下拉选项 |
