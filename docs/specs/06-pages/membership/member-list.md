## 状态机/流转

```
┌──────────┐
│  active  │ (活跃，正常使用会员权益)
└────┬─────┘
     │ 冻结                          过期 (超过 expiry_date)
     ├────────────────┐              │
     ▼                │              ▼
┌──────────┐         │         ┌──────────┐
│  frozen  │         │         │ expired  │
└────┬─────┘         │         └────┬─────┘
     │ 解冻           │              │ 续期
     └──────→ active ←───────────────┘
                │
                │ 注销
                ▼
          ┌──────────────┐
          │  cancelled   │ (已注销，不可恢复)
          └──────────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `active` | `frozen` | 手动点击 [冻结] | 冻结期间会员权益暂停; 积分不过期 |
| `active` | `expired` | 超过 expiry_date | 系统定时任务每日 00:00 扫描; 到期前 30/7/1 天发送提醒 |
| `active` | `cancelled` | 手动注销 | 需二次确认; 储值余额需先退还; 不可恢复 |
| `frozen` | `active` | 手动点击 [解冻] | 恢复正常会员权益 |
| `expired` | `active` | 续期操作 | 延长到期日期; 续期后恢复活跃状态 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求会员列表 (默认筛选: active 状态, 按累计消费降序) |
| 2 | 并行请求: 会员列表 + 汇总统计 + 等级列表 + 币种配置 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 统计卡片使用数字滚动动画加载 |
| 5 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-04: 新增会员

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新增会员] → 打开右侧抽屉 (480px) |
| 2 | 搜索并选择患者; 选中后自动检查是否已有会员卡 |
| 3 | 如患者已有活跃会员卡 → 提示 "该患者已有会员卡 ({card_no})，无法重复开卡" |
| 4 | 选择会员等级 → 下方显示该等级的权益概要 (折扣率/积分倍率/赠送项目) |
| 5 | 卡号自动生成; 入会日期默认今天; 到期日期默认 +1 年 |
| 6 | 点击 [保存] → 校验表单; 成功 → Toast "会员卡已开通"; 列表刷新 |
| 7 | 点击 [取消] → 如有修改弹出确认: "有未保存的修改，确定离开？" |

### INT-05: 储值充值

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [充值] → 打开充值弹窗，显示当前余额 |
| 2 | 填写充值金额和支付方式 |
| 3 | 点击 [确认充值] → 调用充值 API |
| 4 | 成功 → Toast "充值成功，当前余额 {币种符号}{金额}"; 行内余额实时更新 |
| 5 | 失败 → Toast 错误提示; 弹窗保持打开 |

### INT-06: 积分调整

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [调整积分] → 打开积分调整弹窗，显示当前积分 |
| 2 | 选择调整类型 (增加/扣减); 填写数量和原因 |
| 3 | 扣减时: 如数量超过当前积分 → 实时提示 "扣减数量不得超过当前积分" |
| 4 | 点击 [确认] → 调用积分调整 API |
| 5 | 成功 → Toast "积分已调整，当前积分 {points}"; 行内积分实时更新 |

### INT-07: 冻结/解冻

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [冻结] → 弹出确认弹窗 (红色警告) |
| 2 | 确认 → 调用冻结 API |
| 3 | 成功 → 状态 Badge 动画更新为 "已冻结"; Toast "{姓名}的会员卡已冻结" |
| 4 | 点击 [解冻] → 直接调用解冻 API (无需确认) |
| 5 | 成功 → 状态恢复为 "活跃"; Toast "{姓名}的会员卡已解冻" |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 |
| 4 | 文件名格式: `会员列表_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 大数据量 (>1000条) 时显示导出进度条 |
| 6 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取会员列表 | GET | `/api/v1/member-cards` | `keyword`, `level_id`, `status`, `stored_currency`, `join_date_start`, `join_date_end`, `expiry_date_start`, `expiry_date_end`, `points_min`, `points_max`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取会员卡列表 |
| 获取会员详情 | GET | `/api/v1/member-cards/:id` | - | 获取单个会员卡完整信息 (含患者信息、等级信息) |
| 获取会员汇总 | GET | `/api/v1/member-cards/summary` | 同列表筛选参数 | 统计卡片数据 |
| 新增会员卡 | POST | `/api/v1/member-cards` | `patient_id`, `level_id`, `card_no`, `join_date`, `expiry_date`, `stored_currency`, `stored_balance`, `total_points`, `notes` | 为患者开通会员卡 |
| 储值充值 | POST | `/api/v1/member-cards/:id/recharge` | `amount`, `currency`, `payment_method`, `transaction_id`, `notes` | 储值充值; 创建 StoredValueTransaction |
| 积分调整 | POST | `/api/v1/member-cards/:id/adjust-points` | `type`(increase/decrease), `points`, `description` | 手动调整积分; 创建 PointsTransaction (source=manual_adjust) |
| 冻结会员 | PATCH | `/api/v1/member-cards/:id/freeze` | - | 状态变更为 frozen |
| 解冻会员 | PATCH | `/api/v1/member-cards/:id/unfreeze` | - | 状态恢复为 active |
| 批量冻结 | PATCH | `/api/v1/member-cards/batch-freeze` | `ids[]` | 批量冻结 |
| 批量解冻 | PATCH | `/api/v1/member-cards/batch-unfreeze` | `ids[]` | 批量解冻 |
| 导出会员 | POST | `/api/v1/member-cards/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
| 获取等级列表 | GET | `/api/v1/member-levels` | - | 会员等级选项列表 |
| 获取币种配置 | GET | `/api/v1/settings/currencies` | - | 支持的币种列表和汇率 |
