## 表单字段

### 新建/编辑等级 (抽屉, 520px)

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 等级名称 | `level_name` | Input | 是 | 2-20 字符; 唯一性 | 如: 普通/银卡/金卡/铂金 |
| 等级编码 | `level_code` | Input | 是 | 唯一性; 2-20 字符; 仅小写字母/数字/下划线 | 如: normal/silver/gold/platinum; 编辑时不可修改 |
| 等级图标 | `level_icon` | IconPicker | 否 | | 选择 Emoji 或上传自定义图标 |
| 主题颜色 | `color` | ColorPicker | 是 | 合法颜色值 | 等级标识色; 用于标签、卡片顶部条 |
| 消费门槛 | `min_spending` | MoneyInput | 是 | ≥0, 数值 | 达到该累计消费金额自动升级; 最低等级可设为 0 |
| 门槛币种 | - | ReadOnly | - | - | 跟随诊所默认币种; 所有等级统一币种 |
| 折扣率 | `discount_rate` | NumberInput | 是 | 0-100, 数值, 最多1位小数 | 百分比; 0 表示无折扣; 收费时自动应用 |
| 积分倍率 | `points_multiplier` | NumberInput | 是 | 0.1-10.0, 数值, 最多1位小数 | 默认 1.0; 消费获取积分时乘以此倍率 |
| 赠送项目 | `free_items` | DynamicList | 否 | | 可添加多个: 项目名 + 赠送次数; 如 "洗牙 x 2"; 从费用项目中选择 |
| 专属权益 | `privileges` | TagInput | 否 | 每项最多 50 字符 | 输入权益标签; 如 "优先预约"、"生日礼遇"、"专属客服" |
| 排序权重 | `sort_order` | NumberInput | 是 | ≥0, 整数 | 数值越小排序越前; 新增时自动取最大值 +1 |
| 状态 | `status` | RadioGroup | 是 | | active(启用) / disabled(停用); 默认 active |

### 赠送项目动态列表

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 费用项目 | `free_items[].fee_item_id` | FeeItemSelect | 是 | 必选 | 从已有费用项目中搜索选择 |
| 赠送次数 | `free_items[].quantity` | NumberInput | 是 | ≥1, 整数 | 每年赠送次数; 默认 1 |

### 赠送项目编辑交互

| 行为 | 说明 |
|------|------|
| 新增项目 | 点击 [+ 添加赠送项目] 在列表末尾新增一行 |
| 删除项目 | 每行右侧 [🗑] 按钮; 确认后删除 |
| 项目不可重复 | 同一等级内不可选择相同的费用项目 |

## 状态机/流转

```
┌──────────┐
│  active  │ (启用，新会员可选择此等级)
└────┬─────┘
     │ 停用
     ▼
┌──────────┐
│ disabled │ (停用，新会员不可选; 已有会员不受影响)
└────┬─────┘
     │ 启用
     │
     └──────→ active
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `active` | `disabled` | 点击 [停用] | 停用后新增会员时不显示此等级; 已有该等级的会员权益不受影响 |
| `disabled` | `active` | 点击 [启用] | 恢复可选状态 |

### 删除规则

| 条件 | 是否可删除 | 说明 |
|------|------------|------|
| 该等级无关联会员 (member_count = 0) | 可删除 | 物理删除 |
| 该等级有关联会员 (member_count > 0) | 不可删除 | 仅可停用; 删除按钮置灰并 Tooltip 提示 "该等级下有 {N} 位会员，无法删除，请先迁移会员" |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求等级列表 (按 sort_order 升序排列) |
| 2 | 并行请求: 等级列表 + 各等级会员计数 |
| 3 | 卡片使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 底部自动生成等级权益对比表 |

### INT-02: 拖拽排序

| 步骤 | 行为 |
|------|------|
| 1 | 鼠标按住卡片顶部拖拽手柄 (☰ 图标) 开始拖拽 |
| 2 | 拖拽过程中显示占位虚线框和半透明拖拽预览 |
| 3 | 释放后自动更新 sort_order 并调用排序 API |
| 4 | 成功 → 卡片位置动画更新; Toast "排序已更新" |
| 5 | 失败 → 卡片回到原位; Toast 错误提示 |
| 6 | 等级权益对比表同步更新列顺序 |

### INT-03: 新建/编辑等级

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新增等级] 或卡片上 [编辑] → 打开右侧抽屉 (520px) |
| 2 | 新建时: 等级编码手动输入; sort_order 自动取最大值 +1 |
| 3 | 编辑时: 预填现有数据; 等级编码不可修改 |
| 4 | 主题颜色选择器: 提供预设色板 + 自定义 HEX 输入 |
| 5 | 赠送项目: 可动态增删; 搜索费用项目 |
| 6 | 点击 [保存] → 校验表单; 成功 → Toast "等级已保存"; 卡片列表刷新; 对比表更新 |
| 7 | 点击 [取消] → 如有修改弹出确认: "有未保存的修改，确定离开？" |

### INT-04: 消费门槛验证

| 步骤 | 行为 |
|------|------|
| 1 | 保存时校验: 各等级的 min_spending 不可重复 |
| 2 | 校验: 高排序等级的 min_spending 应大于低排序等级 |
| 3 | 不满足 → 表单错误提示 "消费门槛应高于上一等级 ({等级名} ฿{金额})" |

### INT-05: 停用/启用等级

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [停用] → 弹出确认弹窗 |
| 2 | 确认 → 调用状态变更 API |
| 3 | 成功 → 卡片状态 Badge 更新; 卡片视觉变灰; Toast "{等级名}已停用" |
| 4 | 点击 [启用] → 直接调用 API (无需确认) |
| 5 | 成功 → 卡片恢复正常视觉; Toast "{等级名}已启用" |

### INT-06: 删除等级

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [删除] → 弹出红色确认弹窗 |
| 2 | 确认 → 调用删除 API |
| 3 | 成功 → 卡片淡出动画移除; Toast "{等级名}已删除" |
| 4 | 失败 (有关联会员) → Toast "删除失败: 该等级下有 {N} 位会员" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取等级列表 | GET | `/api/v1/member-levels` | `status` | 获取所有等级; 按 sort_order 升序; 含 member_count |
| 获取等级详情 | GET | `/api/v1/member-levels/:id` | - | 获取单个等级完整信息 |
| 创建等级 | POST | `/api/v1/member-levels` | `level_name`, `level_code`, `level_icon`, `color`, `min_spending`, `discount_rate`, `points_multiplier`, `free_items[]`, `privileges[]`, `sort_order`, `status` | 创建会员等级 |
| 更新等级 | PUT | `/api/v1/member-levels/:id` | 同创建参数 (level_code 不可改) | 更新会员等级 |
| 删除等级 | DELETE | `/api/v1/member-levels/:id` | - | 仅无关联会员的等级可删除 |
| 更新排序 | PATCH | `/api/v1/member-levels/sort` | `items[]` (每项含 `id`, `sort_order`) | 批量更新排序; 拖拽后调用 |
| 更新状态 | PATCH | `/api/v1/member-levels/:id/status` | `status` | 启用/停用等级 |
| 获取费用项目列表 | GET | `/api/v1/fee-items` | `status=active`, `keyword` | 赠送项目选择器数据源 |
