## 表单字段

### 积分规则配置 (积分规则 Tab)

#### 获取规则

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 消费获取-启用 | `earn_rules.consumption.enabled` | Switch | - | - | 是否开启消费获取积分; 默认开启 |
| 消费获取-每消费金额 | `earn_rules.consumption.spend_amount` | MoneyInput | 是 (开启时) | >0, 数值 | 每消费多少金额获得积分; 如每消费 ฿100 |
| 消费获取-获得积分 | `earn_rules.consumption.earn_points` | NumberInput | 是 (开启时) | >0, 整数 | 获得积分数; 如 1 分; 实际获得 = earn_points × 等级倍率 |
| 每日签到-启用 | `earn_rules.sign_in.enabled` | Switch | - | - | 是否开启每日签到获取积分; 默认关闭 |
| 每日签到-基础积分 | `earn_rules.sign_in.base_points` | NumberInput | 是 (开启时) | >0, 整数 | 每次签到基础积分; 如 10 分 |
| 每日签到-连续加成 | `earn_rules.sign_in.streak_bonus` | NumberInput | 否 | ≥0, 整数 | 连续签到每天额外增加的积分; 如连续第2天 +5, 第3天 +10; 默认 5 |
| 每日签到-加成上限 | `earn_rules.sign_in.streak_max_days` | NumberInput | 否 | ≥1, 整数 | 连续签到加成的最大天数; 默认 7 |
| 推荐好友-启用 | `earn_rules.referral.enabled` | Switch | - | - | 是否开启推荐好友获取积分; 默认关闭 |
| 推荐好友-推荐人积分 | `earn_rules.referral.referrer_points` | NumberInput | 是 (开启时) | >0, 整数 | 推荐人获得积分 |
| 推荐好友-被推荐人积分 | `earn_rules.referral.referee_points` | NumberInput | 否 | ≥0, 整数 | 被推荐人获得积分; 0 表示不赠送 |
| 推荐好友-触发条件 | `earn_rules.referral.trigger` | Select | 是 (开启时) | 仅限枚举值 | register(注册即送) / first_consumption(首次消费后) |

#### 兑换规则

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 兑换比例-积分数 | `redeem_rules.points_per_unit` | NumberInput | 是 | >0, 整数 | 兑换所需积分数; 如 100 积分 |
| 兑换比例-抵扣金额 | `redeem_rules.value_per_unit` | MoneyInput | 是 | >0, 数值 | 兑换对应金额; 如 ฿10 |
| 兑换比例-币种 | `redeem_rules.currency` | ReadOnly | - | - | 跟随诊所默认币种 |
| 最低兑换积分 | `redeem_rules.min_redeem_points` | NumberInput | 是 | >0, 整数 | 单次最低兑换积分; 如 100 分起 |
| 单次最大兑换 | `redeem_rules.max_redeem_points` | NumberInput | 否 | ≥min_redeem_points, 整数 | 单次最大兑换积分; 0 或空表示不限 |
| 单笔最大抵扣比例 | `redeem_rules.max_discount_percent` | NumberInput | 否 | 0-100, 整数 | 单笔消费最多用积分抵扣的比例; 如 50% 表示最多抵扣消费的一半; 0 或空表示不限 |

#### 过期规则

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 积分有效期 | `expiry_rules.validity_months` | NumberInput | 是 | ≥1, 整数 | 积分获取后的有效月数; 如 12 个月; 过期后自动清零 |
| 是否启用过期 | `expiry_rules.enabled` | Switch | - | - | 关闭则积分永不过期; 默认开启 |
| 过期提醒-提前天数 | `expiry_rules.reminder_days` | NumberInput | 是 (启用时) | ≥1, 整数 | 到期前多少天发送提醒; 如 30 天; 通过 WhatsApp/SMS 发送 |

## 状态机/流转

> 积分规则配置为表单保存，无状态流转。积分交易记录为只读，以下为积分生命周期。

```
┌───────────┐    消费/签到/推荐/促销/手动
│  积分获取   │ ◄──────────────────────
└────┬──────┘
     │
     ▼
┌───────────┐    兑换消费抵扣
│  积分持有   │ ───────────────→ 积分扣减
└────┬──────┘
     │ 超过有效期
     ▼
┌───────────┐
│  积分过期   │ (系统定时任务自动处理)
└───────────┘
```

### 积分流转规则

| 变动类型 | 触发条件 | 说明 |
|----------|----------|------|
| `earn` (获取) | 消费结算/每日签到/推荐成功/活动赠送/手动增加 | points > 0; balance_after = 旧余额 + points |
| `redeem` (兑换) | 收费时积分抵扣 | points < 0; balance_after = 旧余额 - abs(points) |
| `expire` (过期) | 系统定时任务每日 00:00 扫描 | points < 0; 自动清零过期积分; 批量创建过期记录 |
| `adjust` (调整) | 管理员手动调整 | points 可正可负; 需填写原因; 审计追踪 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时默认显示 "积分规则" Tab |
| 2 | 并行请求: 积分规则配置 + 币种配置 |
| 3 | 表单使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL hash 参数 (#rules / #transactions)，如有则切换到对应 Tab |

### INT-02: 积分规则保存

| 步骤 | 行为 |
|------|------|
| 1 | 修改规则字段后，[保存] 按钮高亮变为 Primary 状态 |
| 2 | 点击 [保存] → 校验所有字段 |
| 3 | 校验失败 → 定位到第一个错误字段; 显示错误提示 |
| 4 | 校验成功 → 调用保存 API |
| 5 | 成功 → Toast "积分规则已保存"; [保存] 按钮恢复禁用状态 |
| 6 | 失败 → Toast 错误提示 |
| 7 | 离开页面时如有未保存修改 → 弹出确认: "有未保存的规则修改，确定离开？" |

### INT-03: Tab 切换

| 步骤 | 行为 |
|------|------|
| 1 | 点击 Tab 切换; URL hash 同步更新 |
| 2 | 切换到 "积分流水" Tab → 自动加载流水列表 (默认按时间降序) |
| 3 | 切换到 "积分规则" Tab → 显示已保存的规则配置 |
| 4 | 如 "积分规则" Tab 有未保存修改 → 切换时弹出确认 |

### INT-04: 积分流水搜索

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-05: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-06: 类型-来源联动筛选

| 步骤 | 行为 |
|------|------|
| 1 | 选择类型后，来源下拉框仅显示该类型对应的来源选项 |
| 2 | 选择 "全部" 类型 → 来源显示所有选项 |
| 3 | 切换类型时，如已选来源不在新选项中 → 自动重置来源为 "全部" |

### INT-07: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 导出当前筛选条件下的全部积分流水 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 |
| 4 | 文件名格式: `积分流水_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 大数据量 (>1000条) 时显示导出进度条 |
| 6 | 导出完成 → Toast 提示 "导出成功" |

### INT-08: 开关联动

| 步骤 | 行为 |
|------|------|
| 1 | 关闭某获取规则开关 (如签到) → 下方对应字段置灰禁用 |
| 2 | 开启开关 → 字段恢复可编辑; 显示上次保存的值 |
| 3 | 关闭过期规则 → 过期提醒和冻结过期字段隐藏 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取积分规则 | GET | `/api/v1/points/rules` | - | 获取当前积分规则配置 |
| 保存积分规则 | PUT | `/api/v1/points/rules` | `earn_rules`, `redeem_rules`, `expiry_rules` | 保存/更新积分规则配置 |
| 获取积分流水 | GET | `/api/v1/points/transactions` | `keyword`, `type`, `source`, `date_start`, `date_end`, `points_min`, `points_max`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取积分流水列表 |
| 导出积分流水 | POST | `/api/v1/points/transactions/export` | 同列表筛选参数 | 导出 Excel 文件 |
| 获取币种配置 | GET | `/api/v1/settings/currencies` | - | 支持的币种列表和汇率 |
