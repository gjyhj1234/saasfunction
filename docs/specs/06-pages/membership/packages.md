## 状态机/流转

```
┌──────────┐
│  active  │ (在售，可被会员购买)
└────┬─────┘
     │ 下架 / 售罄 (sold_count ≥ max_sell_count)
     ├──────────────────────┐
     ▼                      ▼
┌──────────┐          ┌──────────┐
│ disabled │          │ sold_out │ (自动触发; sold_count ≥ max_sell_count)
└────┬─────┘          └────┬─────┘
     │ 上架                 │ 下架
     │                      │
     └──────→ active        └──────→ disabled
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `active` | `disabled` | 手动点击 [下架] | 下架后不可购买; 已购买的套餐实例不受影响 |
| `active` | `sold_out` | sold_count ≥ max_sell_count | 系统自动触发; 仅限设置了 max_sell_count 的套餐 |
| `disabled` | `active` | 手动点击 [上架] | 恢复在售状态; 可被会员购买 |
| `sold_out` | `disabled` | 手动点击 [下架] | 从售罄变为停用 |
| `sold_out` | `active` | 增加 max_sell_count 后手动上架 | 需先编辑增加限售数量; 然后上架 |

### 删除规则

| 条件 | 是否可删除 | 说明 |
|------|------------|------|
| 未被任何会员购买 (sold_count = 0) | 可删除 | 物理删除 |
| 已被会员购买 (sold_count > 0) | 不可删除 | 仅可下架; 删除按钮置灰并 Tooltip 提示 "该套餐已售出 {N} 份，无法删除" |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求套餐列表 (默认按创建时间降序) |
| 2 | 并行请求: 套餐列表 + 汇总统计 + 等级列表 + 币种配置 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 统计卡片使用数字滚动动画加载 |
| 5 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-04: 新建/编辑套餐

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建套餐] 或 [编辑] → 打开右侧抽屉 (640px) |
| 2 | 编辑时: 预填现有数据; 包含项目列表预填 |
| 3 | 包含项目区域: 搜索并选择费用项目; 设置使用次数 |
| 4 | 选择费用项目后自动填充项目名称和单价 |
| 5 | 实时计算: 项目原价合计 + 总使用次数; 显示折扣率 |
| 6 | 售价填写时: 如 > original_price → 实时提示 "售价不能高于原价" |
| 7 | 点击 [保存] → 校验表单; 检查包含项目 ≥ 1 项 |
| 8 | 成功 → Toast "套餐已保存"; 列表刷新; 统计更新 |
| 9 | 点击 [取消] → 如有修改弹出确认: "有未保存的修改，确定离开？" |

### INT-05: 复制套餐

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [复制] → 打开新建抽屉并预填数据 |
| 2 | 套餐名称自动追加 "(副本)" |
| 3 | 状态默认为 disabled (需手动上架) |
| 4 | sold_count 重置为 0 |

### INT-06: 上架/下架

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [下架] → 弹出确认弹窗 |
| 2 | 确认 → 调用状态变更 API |
| 3 | 成功 → 行内状态 Badge 动画更新; Toast "{套餐名}已下架" |
| 4 | 点击 [上架] → 直接调用 API (无需确认) |
| 5 | 成功 → 状态恢复为 "在售"; Toast "{套餐名}已上架" |

### INT-07: 售罄自动处理

| 步骤 | 行为 |
|------|------|
| 1 | 会员购买套餐后 sold_count +1 |
| 2 | 如 sold_count ≥ max_sell_count → 自动变更为 sold_out |
| 3 | 套餐列表实时刷新; 状态 Badge 变为 "已售罄" (橙色) |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 + 包含项目明细 |
| 4 | 文件名格式: `会员套餐_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取套餐列表 | GET | `/api/v1/member-packages` | `keyword`, `status`, `applicable_levels`, `price_min`, `price_max`, `currency`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取套餐列表 |
| 获取套餐详情 | GET | `/api/v1/member-packages/:id` | - | 获取单个套餐完整信息 (含包含项目明细) |
| 获取套餐汇总 | GET | `/api/v1/member-packages/summary` | 同列表筛选参数 | 统计卡片数据 |
| 创建套餐 | POST | `/api/v1/member-packages` | `package_name`, `description`, `original_price`, `selling_price`, `currency`, `validity_days`, `applicable_levels[]`, `max_sell_count`, `included_items[]`, `status` | 创建会员套餐 |
| 更新套餐 | PUT | `/api/v1/member-packages/:id` | 同创建参数 | 更新会员套餐 |
| 删除套餐 | DELETE | `/api/v1/member-packages/:id` | - | 仅 sold_count = 0 的套餐可删除 |
| 批量更新状态 | PATCH | `/api/v1/member-packages/batch-status` | `ids[]`, `status` | 批量上架/下架 |
| 导出套餐 | POST | `/api/v1/member-packages/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
| 获取等级列表 | GET | `/api/v1/member-levels` | `status=active` | 适用等级选择器数据源 |
| 获取费用项目列表 | GET | `/api/v1/fee-items` | `status=active`, `keyword` | 包含项目选择器数据源 |
| 获取币种配置 | GET | `/api/v1/settings/currencies` | - | 支持的币种列表和汇率 |
