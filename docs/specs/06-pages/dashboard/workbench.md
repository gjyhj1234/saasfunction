## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时显示骨架屏（卡片和区块占位） |
| 2 | 并行请求所有统计 API |
| 3 | 各区块独立渲染，先返回先显示 |
| 4 | 全部加载完成后隐藏骨架屏 |
| 5 | API 失败的区块显示 "加载失败，点击重试" |

### INT-02: 自动刷新

| 属性 | 规范 |
|------|------|
| 刷新间隔 | 60 秒 |
| 刷新范围 | 统计卡片 + 预约时间线 + 待办事项 |
| 刷新方式 | 静默刷新，无 loading 状态（避免闪烁） |
| 手动刷新 | 页面右上角提供刷新按钮，点击立即刷新所有数据 |
| 暂停刷新 | 页面不在前台时暂停自动刷新 (visibilitychange 事件) |
| WebSocket | 预约状态变更通过 WebSocket 推送实时更新 |

### INT-03: 卡片点击

| 交互 | 行为 |
|------|------|
| 点击统计卡片 | 跳转到对应模块列表页（带默认筛选条件） |
| 点击卡片内子指标 | 跳转到对应模块列表页（带更精确的筛选条件） |
| Hover | 卡片轻微上浮，阴影加深 |
| 无权限 | 卡片显示但数据显示 "--"，点击无反应 |

### INT-04: 响应式布局

| 断点 | 布局 |
|------|------|
| >= 1280px | 6 列统计卡片；内容区 60% + 40% 两列 |
| 768-1279px | 3 列统计卡片（2 行）；内容区单列堆叠 |
| < 768px | 2 列统计卡片（3 行）；内容区单列堆叠 |

### INT-05: 角色差异化显示

| 角色 | 差异 |
|------|------|
| 管理员/院长 | 显示全部卡片和区块，数据为全诊所汇总 |
| 医生 | 预约时间线只显示自己的预约，患者统计为自己的患者 |
| 前台 | 重点显示预约和收费相关卡片，隐藏库存/会员管理 |
| 仓管 | 重点显示库存预警，隐藏营收/会员相关卡片 |

## API 数据源

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 今日预约统计 | GET | `/api/dashboard/appointment-stats` | 参数: `date` |
| 营收统计 | GET | `/api/dashboard/revenue-stats` | 参数: `dateRange` (today/week/month/custom) |
| 患者统计 | GET | `/api/dashboard/patient-stats` | 参数: `dateRange` |
| 欠费汇总 | GET | `/api/dashboard/overdue-summary` | — |
| 会员统计 | GET | `/api/dashboard/membership-stats` | — |
| 库存预警汇总 | GET | `/api/dashboard/stock-alert-summary` | — |
| 今日预约列表 | GET | `/api/dashboard/today-appointments` | 返回时间线所需数据 |
| 待办事项 | GET | `/api/dashboard/todo-list` | 聚合各模块待办 |
| 商城待处理订单 | GET | `/api/dashboard/pending-mall-orders` | 参数: `limit=5` |

