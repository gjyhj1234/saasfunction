## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求影像列表 (默认按拍摄日期降序) |
| 2 | 并行请求: 影像列表 + 医生下拉 + 影像类型枚举 |
| 3 | 网格视图使用骨架卡片作为加载占位; 列表视图使用骨架屏 |
| 4 | 检查 URL query 参数: `patient_id` / `tooth_number` / `view` (grid/list) |
| 5 | 缩略图懒加载 (Intersection Observer), 进入视口后加载 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时显示 loading 状态 |
| 3 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 视图切换

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [网格] / [列表] 切换按钮 |
| 2 | 保持当前筛选条件和页码 |
| 3 | 视图类型持久化到 localStorage |
| 4 | 网格视图: 自适应列数 (1280px=4列, 1024px=3列, 768px=2列, <768px=1列) |

### INT-04: 影像上传

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [上传影像] 打开上传抽屉 (480px) |
| 2 | 拖拽或点击上传区域选择文件; 支持多文件 |
| 3 | 显示每个文件的上传进度条 |
| 4 | 填写影像信息 (类型/日期/患者) |
| 5 | **牙位绑定为必填** → 弹出牙位图选择关联牙位 |
| 6 | 可选标记术前/术后 |
| 7 | 点击 [确认上传] → 上传文件并保存信息 |
| 8 | 成功 → Toast "已上传 {N} 张影像"; 列表自动刷新 |
| 9 | 失败 → 显示失败文件列表, 可重试 |

### INT-05: 牙位绑定

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [绑定牙位] 弹出牙位绑定弹窗 |
| 2 | 显示交互式牙位图 (同牙位图页面简化版) |
| 3 | 点击牙齿选中/取消; 已绑定牙位蓝色高亮预选中 |
| 4 | 区域自动推断: 选中同一象限 → 自动设为对应象限; 多象限 → full_mouth |
| 5 | 点击 [确认] → 保存绑定关系; Toast "已绑定 {N} 颗牙位" |

### INT-06: 术前术后对比

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [术前术后对比] 进入对比模式 |
| 2 | 选择第 1 张影像 (点击或从已选中影像进入) |
| 3 | 如有 `paired_image_id` → 自动加载配对影像 |
| 4 | 如无配对 → 右侧面板展示同患者同牙位影像供选择 |
| 5 | 左右面板分别显示; 支持同步缩放/拖动/旋转 |
| 6 | 点击 [退出对比] 返回列表/网格 |

### INT-07: 影像查看

| 步骤 | 行为 |
|------|------|
| 1 | 网格: 点击缩略图; 列表: 点击缩略图或 [查看] |
| 2 | 打开全屏影像查看器 |
| 3 | 加载原始大图 + 标注层 |
| 4 | 右侧面板显示影像信息/关联牙位/标注列表/配对影像 |
| 5 | 键盘 ← → 切换上一张/下一张; ESC 关闭 |

### INT-08: 影像标注

| 步骤 | 行为 |
|------|------|
| 1 | 在查看器中点击 [标注✏️] 进入标注模式 |
| 2 | 左侧工具栏显示标注工具 (箭头/矩形/圆形/自由线/文字/测量) |
| 3 | 选择工具后在影像上绘制标注 |
| 4 | 每条标注可设置颜色和文字说明 |
| 5 | 点击 [保存标注] → 保存标注数据; Toast "标注已保存" |
| 6 | 标注层可切换显示/隐藏 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取影像列表 | GET | `/api/v1/clinical-images` | `keyword`, `patient_id`, `image_type[]`, `region`, `tooth_number`, `capture_date_start`, `capture_date_end`, `is_before_treatment`, `doctor_id`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取影像列表 (含缩略图) |
| 获取影像详情 | GET | `/api/v1/clinical-images/:id` | - | 含完整信息 + 标注 + 牙位关联 + 配对影像 |
| 上传影像 | POST | `/api/v1/clinical-images` | `file` (multipart), `patient_id`, `image_type`, `capture_date`, `appointment_id`, `treatment_record_id`, `is_before_treatment`, `region`, `notes` | 上传影像文件并保存信息 |
| 批量上传影像 | POST | `/api/v1/clinical-images/batch` | `files[]` (multipart), 同上公共参数 | 批量上传多张影像 |
| 更新影像信息 | PUT | `/api/v1/clinical-images/:id` | `image_type`, `capture_date`, `is_before_treatment`, `notes` | 更新影像基本信息 |
| 删除影像 | DELETE | `/api/v1/clinical-images/:id` | - | 删除影像及其标注和关联 |
| 批量删除影像 | DELETE | `/api/v1/clinical-images/batch` | `ids[]` | 批量删除 |
| 绑定牙位 | POST | `/api/v1/clinical-images/:id/tooth-links` | `tooth_numbers[]`, `region` | 绑定影像到牙位 |
| 更新牙位绑定 | PUT | `/api/v1/clinical-images/:id/tooth-links` | `tooth_numbers[]`, `region` | 替换牙位绑定 |
| 保存标注 | PUT | `/api/v1/clinical-images/:id/annotations` | `annotations[]` | 保存影像标注数据 |
| 设置配对 | POST | `/api/v1/clinical-images/:id/pair` | `paired_image_id` | 设置术前术后配对关系 (双向) |
| 取消配对 | DELETE | `/api/v1/clinical-images/:id/pair` | - | 取消配对关系 |
| 批量下载 | POST | `/api/v1/clinical-images/download` | `ids[]` | 打包为 ZIP 下载 |
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active` | 医生下拉选项 |
