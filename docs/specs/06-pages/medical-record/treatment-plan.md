## 行操作

| 序号 | 按钮文本 | 操作类型 | 图标 | 权限 | 确认弹窗 | 可见条件 | 说明 |
|------|----------|----------|------|------|----------|----------|------|
| 1 | 查看 | 链接 | 👁 | `treatment_plan:view` | 否 | 始终可见 | 打开治疗方案详情抽屉 |
| 2 | 编辑 | 链接 | ✏️ | `treatment_plan:edit` | 否 | 仅 `draft` / `proposed` 状态 | 打开编辑方案抽屉 |
| 3 | 提交审批 | 操作 | 📤 | `treatment_plan:edit` | 是: "确认提交方案给患者确认？" | 仅 `draft` 状态 | 状态变更为 `proposed` |
| 4 | 记录同意 | 操作 | ✅ | `treatment_plan:edit` | 是: "确认患者已同意此方案？" | 仅 `proposed` 状态且未同意 | 记录患者同意，状态变更为 `accepted` |
| 5 | 开始执行 | 操作 | ▶️ | `treatment_plan:edit` | 否 | 仅 `accepted` 状态 | 状态变更为 `in_progress` |
| 6 | 暂停 | 操作 | ⏸ | `treatment_plan:edit` | 是: "确认暂停此方案？" | 仅 `in_progress` 状态 | 状态变更为 `suspended` |
| 7 | 恢复 | 操作 | ▶️ | `treatment_plan:edit` | 否 | 仅 `suspended` 状态 | 状态变更为 `in_progress` |
| 8 | 取消 | 危险操作 | ❌ | `treatment_plan:edit` | 是(需填原因): "确认取消此方案？此操作不可撤销" | 非 `completed` / `cancelled` 状态 | 状态变更为 `cancelled`; 红色文字 |

### 行操作显示规则

| 条件 | 显示按钮 |
|------|----------|
| 超过 3 个可见按钮 | 前 2 个直接显示，其余收入 "更多" 下拉菜单 |
| 无任何可操作按钮 | 仅显示 "查看" |

## 状态机/流转

```
┌──────────┐
│  draft   │ (草案，医生编辑中)
└────┬─────┘
     │ 提交审批
     ▼
┌──────────┐
│ proposed │ (已提交，待患者确认)
└────┬─────┘
     │ 患者同意                         ┌──────────┐
     ▼                                  │cancelled │
┌──────────┐   ←── 任何非 completed ───→│          │
│ accepted │       状态均可取消          └──────────┘
└────┬─────┘
     │ 开始执行
     ▼
┌──────────┐          ┌──────────┐
│in_progress│←────────│suspended │
└────┬──┬──┘  恢复    └──────────┘
     │  │                  ▲
     │  └── 暂停 ──────────┘
     │ 所有步骤完成
     ▼
┌──────────┐
│completed │
└──────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `draft` | `proposed` | 点击 [提交审批] | 需至少 1 个步骤 |
| `draft` | `cancelled` | 点击 [取消] | |
| `proposed` | `accepted` | 点击 [记录同意] | 记录 consent_date |
| `proposed` | `draft` | 点击 [退回修改] | 退回医生重新编辑 |
| `proposed` | `cancelled` | 点击 [取消] | 患者拒绝 |
| `accepted` | `in_progress` | 点击 [开始执行] | 自动记录 start_date |
| `accepted` | `cancelled` | 点击 [取消] | |
| `in_progress` | `suspended` | 点击 [暂停] | 需填暂停原因 |
| `in_progress` | `completed` | 所有步骤完成或跳过 | 自动流转 |
| `in_progress` | `cancelled` | 点击 [取消] | 需填取消原因 |
| `suspended` | `in_progress` | 点击 [恢复] | |
| `suspended` | `cancelled` | 点击 [取消] | |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求治疗方案列表 (默认按创建时间降序) |
| 2 | 并行请求: 方案列表 + 医生下拉 + 币种配置 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有 `patient_id` 则自动筛选该患者方案 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-04: 新建方案

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建方案] 打开右侧抽屉 (640px) |
| 2 | 如从患者详情页进入, `patient_id` 自动预填 |
| 3 | 填写基本信息 + 动态添加治疗步骤 |
| 4 | 牙位字段点击弹出迷你牙位图弹窗选择 |
| 5 | 步骤支持拖拽排序、添加和删除 |
| 6 | 点击 [保存草案] → 保存为 draft 状态; Toast "方案已保存" |
| 7 | 点击 [保存并提交] → 保存并直接变更为 proposed 状态 |

### INT-05: 查看详情

| 步骤 | 行为 |
|------|------|
| 1 | 点击方案编号/名称或 [查看] 按钮，打开详情抽屉 (640px) |
| 2 | 加载方案完整信息 + 所有步骤 |
| 3 | 进度条动画显示当前进度 |
| 4 | 当前执行中步骤高亮显示 (黄色背景) |
| 5 | 底部显示可用操作按钮 (根据状态动态显示) |

### INT-06: 完成步骤

| 步骤 | 行为 |
|------|------|
| 1 | 在详情抽屉中点击 [完成步骤] |
| 2 | 弹窗填写实际费用和日期 |
| 3 | 确认后当前步骤状态变更为 `completed` |
| 4 | 进度条更新; 如所有步骤完成则方案自动变为 `completed` |
| 5 | Toast "步骤 #{step_number} 已完成" |

### INT-07: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx) |
| 4 | 文件名格式: `治疗方案_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 大数据量 (>1000条) 时显示导出进度条 |
| 6 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取方案列表 | GET | `/api/v1/treatment-plans` | `keyword`, `status[]`, `doctor_id`, `patient_id`, `currency`, `cost_min`, `cost_max`, `start_date`, `end_date`, `consent`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取治疗方案列表 |
| 获取方案详情 | GET | `/api/v1/treatment-plans/:id` | - | 含基本信息 + 所有步骤 + 费用汇总 |
| 创建方案 | POST | `/api/v1/treatment-plans` | `patient_id`, `doctor_id`, `plan_name`, `description`, `total_estimated_cost`, `currency`, `start_date`, `estimated_end_date`, `steps[]` | 创建治疗方案 (含步骤) |
| 更新方案 | PUT | `/api/v1/treatment-plans/:id` | 同创建参数 | 更新方案基本信息和步骤 |
| 更新方案状态 | PATCH | `/api/v1/treatment-plans/:id/status` | `status`, `reason` | 状态变更操作 |
| 记录患者同意 | POST | `/api/v1/treatment-plans/:id/consent` | `consent_date` | 记录患者同意 |
| 完成步骤 | PATCH | `/api/v1/treatment-plans/:id/steps/:step_id/complete` | `actual_cost`, `actual_date`, `note` | 完成某个步骤 |
| 调整步骤 | PUT | `/api/v1/treatment-plans/:id/steps` | `steps[]` | 批量更新步骤 (增删改排序) |
| 导出方案 | POST | `/api/v1/treatment-plans/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
| 获取医生列表 | GET | `/api/v1/doctors` | `status=active` | 医生下拉选项 |
