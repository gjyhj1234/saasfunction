### 牙面标识 (Five-Surface System)

| 缩写 | 英文 | 中文 | 说明 |
|------|------|------|------|
| M | Mesial | 近中面 | 靠近中线的一面 |
| D | Distal | 远中面 | 远离中线的一面 |
| B | Buccal | 颊面 | 朝向脸颊的一面 |
| L | Lingual | 舌面 | 朝向舌头的一面 |
| O | Occlusal | 咬合面 | 咬合面 (前牙为切缘 Incisal) |

### 牙面状态值

| 状态值 | 中文 | 颜色 | 说明 |
|--------|------|------|------|
| `normal` | 正常 | 无标记 | 默认状态 |
| `decayed` | 龋坏 | #EF4444 (红) | 该面有龋坏 |
| `filled` | 充填 | #3B82F6 (蓝) | 该面已充填 |
| `crown_covered` | 冠覆盖 | #8B5CF6 (紫) | 被牙冠覆盖 |

### 基本状态字段

| 字段标签 | 字段名 | 组件类型 | 必填 | 验证规则 | 说明 |
|----------|--------|----------|------|----------|------|
| 当前状态 | `current_status` | Select | 是 | 仅限枚举值 | 下拉选择牙齿状态，切换后颜色实时变化 |
| 松动度 | `mobility_grade` | Select | 否 | 0-3 整数 | 0=无松动, 1=轻微, 2=中度, 3=重度 |
| 牙周袋深度 | `periodontal_pocket_depth` | NumberInput | 否 | 0-15, 单位 mm | 支持小数点后 1 位 |
| 备注 | `note` | Textarea | 否 | 最多 500 字符 | 针对该牙齿的备注信息 |

## 筛选器

| 序号 | 筛选项 | 组件类型 | 选项 | 默认值 | 说明 |
|------|--------|----------|------|--------|------|
| 1 | 状态筛选 | MultiSelect | healthy/decayed/missing/filled/crowned/bridged/implant/root_canal/extracted/impacted/fractured | 全部 | 选择后仅高亮显示匹配状态的牙齿，其余灰化 |
| 2 | 象限筛选 | CheckboxGroup | Q1(右上) / Q2(左上) / Q3(左下) / Q4(右下) / 全部 | 全部 | 选择后仅高亮对应象限 |
| 3 | 齿列类型 | RadioGroup | 成人恒牙(32颗) / 儿童乳牙(20颗) | 根据患者年龄自动选择 (≥13 岁默认成人, <6 岁默认儿童, 6-12 岁提示医生确认) | 切换后重新渲染牙位图 |

### 筛选器交互

| 行为 | 说明 |
|------|------|
| 切换筛选项 | 即时筛选，无需点击查询按钮 |
| 高亮逻辑 | 匹配的牙齿正常显示，不匹配的牙齿半透明 (opacity: 0.3) |
| 统计更新 | 筛选后底部状态统计同步更新为筛选范围内的计数 |

### 状态流转规则

| 当前状态 | 可流转至 | 说明 |
|----------|----------|------|
| `healthy` | `decayed`, `impacted`, `fractured`, `missing` | 发现问题或因外伤缺失 |
| `decayed` | `filled`, `root_canal`, `extracted`, `crowned` | 治疗处理 |
| `filled` | `decayed`, `crowned`, `healthy` | 充填物脱落/升级/恢复 |
| `root_canal` | `crowned`, `extracted` | 根管后通常做冠修复 |
| `crowned` | `bridged`, `extracted`, `fractured` | 修复体损坏或需拔除 |
| `bridged` | `missing` | 桥体失败 |
| `missing` | `implant`, `bridged` | 种植或桥修复 |
| `implant` | `healthy`, `fractured` | 种植成功/种植体失败 |
| `impacted` | `extracted`, `healthy` | 拔除阻生齿或自行萌出 |
| `extracted` | `missing` | 拔除后为缺失状态 |
| `fractured` | `crowned`, `extracted` | 修复或拔除 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 通过 `patient_id` 请求患者基本信息和牙位图数据 |
| 2 | 并行请求: 患者信息 + 牙位图数据 + 牙齿记录列表 |
| 3 | 根据患者年龄自动选择齿列类型 (≥13 岁=成人 32 颗, <6 岁=儿童 20 颗, 6-12 岁弹窗提示医生选择齿列类型) |
| 4 | 渲染 SVG 牙位图，每颗牙齿按 `current_status` 上色 |
| 5 | 初始无选中牙齿，右侧面板显示牙位概览统计 |
| 6 | 如无牙位图数据 → 自动创建初始牙位图 (所有牙齿默认 healthy) |

### INT-02: 牙齿点击交互

| 步骤 | 行为 |
|------|------|
| 1 | 鼠标悬停 → 牙齿轮廓高亮 + Tooltip 显示 "{FDI编号} {牙齿名称} {当前状态}" |
| 2 | 点击牙齿 → 选中状态 (蓝色外框) + 右侧面板加载该牙齿详情 |
| 3 | 右侧面板显示: 基本状态 / 牙面状态图 / 历史时间线 / 关联影像 |
| 4 | 点击其他牙齿 → 切换选中; 如有未保存修改弹窗提示 "是否保存当前修改？" |
| 5 | 点击空白区域 → 取消选中, 右侧面板回到概览统计 |

### INT-03: 牙齿状态修改

| 步骤 | 行为 |
|------|------|
| 1 | 在右侧面板点击 [编辑] 进入编辑模式 |
| 2 | 修改状态下拉选择 → 牙位图上对应牙齿实时变色预览 |
| 3 | 修改牙面状态 → 点击牙面图对应区域或下拉选择 |
| 4 | 点击 [保存] → 调用更新 API; 成功后 Toast "牙齿 #{tooth_number} 状态已更新" |
| 5 | 点击 [取消] → 恢复修改前状态 |
| 6 | 状态变更自动记录到历史时间线 |

### INT-04: 批量更新

| 步骤 | 行为 |
|------|------|
| 1 | 点击工具栏 [批量更新] 进入多选模式 |
| 2 | 鼠标变为十字选择器，点击牙齿逐个选中 (蓝色高亮) |
| 3 | 顶部显示 "已选 {N} 颗" + 状态选择下拉 + [应用] + [取消] |
| 4 | 选择目标状态后点击 [应用] → 确认弹窗 → 调用批量 API |
| 5 | 成功 → 所有选中牙齿颜色更新; Toast "已更新 {N} 颗牙齿状态" |
| 6 | 按 ESC 退出批量模式 |

### INT-05: 齿列切换

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [切换齿列] 下拉，选择成人/儿童 |
| 2 | 如当前有未保存修改 → 弹窗确认 |
| 3 | 切换后重新渲染牙位图 (32 颗 ↔ 20 颗) |
| 4 | 保留已有牙齿数据，新增牙位默认 healthy |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取牙位图 | GET | `/api/v1/dental-charts` | `patient_id` | 获取患者牙位图 (含齿列类型) |
| 创建牙位图 | POST | `/api/v1/dental-charts` | `patient_id`, `chart_type` | 创建初始牙位图 (所有牙齿 healthy) |
| 获取牙齿记录列表 | GET | `/api/v1/dental-charts/:chart_id/teeth` | `quadrant`, `status[]` | 获取牙位图下所有牙齿记录 |
| 获取单颗牙齿详情 | GET | `/api/v1/dental-charts/:chart_id/teeth/:tooth_number` | - | 含牙面状态、松动度、牙周袋深度 |
| 更新牙齿状态 | PATCH | `/api/v1/dental-charts/:chart_id/teeth/:tooth_number` | `current_status`, `surface_conditions`, `mobility_grade`, `periodontal_pocket_depth`, `note` | 更新单颗牙齿 |
| 批量更新牙齿 | PATCH | `/api/v1/dental-charts/:chart_id/teeth/batch` | `tooth_numbers[]`, `current_status`, `surface_conditions` | 批量更新多颗牙齿状态 |
| 获取牙齿历史 | GET | `/api/v1/dental-charts/:chart_id/teeth/:tooth_number/history` | `page`, `page_size` | 牙齿状态变更历史时间线 |
| 获取关联影像 | GET | `/api/v1/clinical-images` | `tooth_number`, `patient_id` | 获取与该牙齿关联的影像 |
| 导出牙位图 PDF | POST | `/api/v1/dental-charts/:chart_id/export` | `format=pdf` | 导出牙位图报告 |
