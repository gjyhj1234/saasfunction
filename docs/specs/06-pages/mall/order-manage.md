## 状态机/流转

```
┌──────────┐
│ pending  │ (待付款)
└────┬─────┘
     │
   ┌─┴───────────────────────┐
   │ 支付成功                  │ 超时/取消
   ▼                          ▼
┌──────────┐            ┌───────────┐
│   paid   │ (已支付)    │ cancelled │ (已取消)
└────┬─────┘            └───────────┘
     │
   ┌─┴───────────────────────┐
   │ 发货                     │ 退款
   ▼                          ▼
┌──────────┐            ┌───────────┐
│ shipping │ (配送中)    │ refunding │ (退款中)
└────┬─────┘            └─────┬─────┘
     │                        │ 退款到账
   ┌─┴───────────┐           ▼
   │ 签收          │ 退款  ┌───────────┐
   ▼              ▼      │ refunded  │ (已退款)
┌──────────┐  ┌────────┐ └───────────┘
│delivered │  │refunding│
│ (已签收)  │  │(退款中) │
└────┬─────┘  └────┬───┘
     │              │ 退款到账
     │ 确认完成      ▼
     ▼         ┌───────────┐
┌──────────┐   │ refunded  │
│completed │   │ (已退款)   │
│ (已完成)  │   └───────────┘
└──────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `pending` | `paid` | 支付回调确认 | 支付网关回调确认支付成功 |
| `pending` | `cancelled` | 超时自动 / 手动取消 | 超时: 30分钟未支付自动取消; 手动: 用户或管理员取消; 取消后自动恢复库存 |
| `paid` | `shipping` | 管理员发货 | 填写快递信息后流转; 需运单号 |
| `paid` | `refunding` | 管理员发起退款 | 未发货退款 |
| `shipping` | `delivered` | 物流签收 / 管理员确认 | 物流回调自动签收或管理员手动确认 |
| `shipping` | `refunding` | 管理员发起退款 | 配送中退款; 需同时处理物流退回 |
| `delivered` | `completed` | 自动完成 / 用户确认 | 签收 7 天后自动完成; 或用户手动确认 |
| `delivered` | `refunding` | 管理员发起退款 | 签收后退款; 需用户退回商品 |
| `refunding` | `refunded` | 退款到账确认 | 支付网关退款回调确认 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求订单列表 (默认无状态筛选, 显示全部) |
| 2 | 并行请求: 订单列表 + 订单状态统计 + 枚举数据 (支付方式/配送方式/快递公司) |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 状态统计栏显示各状态订单数量 |
| 5 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 支持按订单编号、购买人姓名、电话号码搜索 |
| 3 | 搜索时表格显示 loading 状态 |
| 4 | 搜索结果高亮匹配文字 |
| 5 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 查看订单详情

| 步骤 | 行为 |
|------|------|
| 1 | 点击订单编号 / [详情] → 打开订单详情抽屉 (从右侧滑入, 720px) |
| 2 | 抽屉标题: "订单详情: {订单编号}" |
| 3 | 显示五个区块: 买家信息、订单商品、支付信息、物流信息、状态时间线 |
| 4 | 订单商品表格显示: 商品名称、单价、数量、小计、商品项状态 |
| 5 | 状态时间线: 已完成节点实心圆 ●, 未完成节点空心圆 ○ |
| 6 | 抽屉底部显示可用操作按钮 (根据当前状态动态显示) |

### INT-04: 发货操作

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [发货] → 打开发货表单弹窗 |
| 2 | 弹窗标题: "订单发货: {订单编号}" |
| 3 | 选择快递公司 (下拉选择) |
| 4 | 输入运单号 |
| 5 | 可选填发货备注 |
| 6 | 点击 [确认发货] → 表单验证 → 调用发货 API |
| 7 | 成功 → Toast "发货成功"; 关闭弹窗; 订单状态更新为 shipping; 刷新列表 |
| 8 | 如在详情抽屉中操作: 物流信息区块实时更新 |

### INT-05: 确认收货

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [确认收货] → 弹出确认弹窗 |
| 2 | 确认文案: "确认代客户签收订单 {订单编号}？签收后将进入售后期" |
| 3 | 确认后调用确认收货 API |
| 4 | 成功 → Toast "已确认收货"; 状态更新为 delivered |

### INT-06: 退款操作

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [退款] → 打开退款表单弹窗 |
| 2 | 弹窗标题: "订单退款: {订单编号}" |
| 3 | 退款金额默认填入实付金额 (actual_amount); 支持修改为部分退款 |
| 4 | 选择退款原因; 如选 "其他" 则退款说明变为必填 |
| 5 | 点击 [确认退款] → 表单验证 → 调用退款 API |
| 6 | 成功 → Toast "退款已发起"; 订单状态变为 refunding |
| 7 | 退款到账后系统自动更新为 refunded (支付网关回调) |

### INT-07: 取消订单

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [取消] → 弹出确认弹窗 (危险红色样式) |
| 2 | 确认文案: "确认取消订单 {订单编号}？取消后库存将自动恢复" |
| 3 | 确认后调用取消订单 API |
| 4 | 成功 → Toast "订单已取消"; 库存自动恢复; 状态更新为 cancelled |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 + 订单商品明细 |
| 4 | 文件名格式: `订单列表_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

### INT-09: 批量发货

| 步骤 | 行为 |
|------|------|
| 1 | 勾选多条 paid 状态订单 → 点击 [批量发货] |
| 2 | 打开批量发货弹窗; 两种模式: 统一填写 / Excel 导入 |
| 3 | 统一填写: 选择快递公司 + 逐条输入运单号 |
| 4 | Excel 导入: 下载模板 → 填写订单号+运单号 → 上传 |
| 5 | 提交后逐条处理; 显示处理进度 |
| 6 | 完成后显示结果: 成功 {N} 条, 失败 {M} 条 (附失败原因) |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取订单列表 | GET | `/api/v1/mall/orders` | `keyword`, `status`, `delivery_method`, `payment_method`, `date_from`, `date_to`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取订单列表 |
| 获取订单统计 | GET | `/api/v1/mall/orders/stats` | - | 各状态订单数量统计 |
| 获取订单详情 | GET | `/api/v1/mall/orders/:id` | - | 含订单商品、支付信息、物流信息、状态时间线 |
| 订单发货 | POST | `/api/v1/mall/orders/:id/ship` | `express_company`, `tracking_number`, `ship_note` | 填写物流信息并发货 |
| 确认收货 | PATCH | `/api/v1/mall/orders/:id/deliver` | - | 管理员代确认收货 |
| 发起退款 | POST | `/api/v1/mall/orders/:id/refund` | `refund_amount`, `refund_reason`, `refund_note` | 发起退款申请 |
| 取消订单 | PATCH | `/api/v1/mall/orders/:id/cancel` | - | 取消未支付订单 |
| 批量发货 | POST | `/api/v1/mall/orders/batch/ship` | `items[]: { order_id, express_company, tracking_number }` | 批量发货 |
| 导出订单 | POST | `/api/v1/mall/orders/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
| 获取快递公司列表 | GET | `/api/v1/mall/express-companies` | - | 快递公司枚举列表 |
