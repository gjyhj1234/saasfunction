## 状态机/流转

```
┌──────────┐
│  draft   │ (草稿，可编辑/删除)
└────┬─────┘
     │ 提交
     ▼
┌──────────┐
│submitted │ (已提交，等待审批)
└────┬─────┘
     │ 审批通过              审批驳回
     ▼                      ▼
┌──────────┐          ┌──────────┐
│ approved │          │  draft   │ (退回草稿)
└────┬─────┘          └──────────┘
     │ 首次收货
     ▼
┌──────────┐
│receiving │ (收货中，部分收货)
└────┬─────┘
     │ 全部收货完成
     ▼
┌──────────┐
│completed │ (已完成)
└──────────┘

--- 取消分支 ---
draft ──────→ cancelled
submitted ──→ cancelled
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `draft` | `submitted` | 点击 [提交] | 验证至少有1项产品; 验证供应商和仓库 |
| `draft` | `cancelled` | 点击 [取消] | 需确认弹窗 |
| `submitted` | `approved` | 审批人点击 [通过] | 需 `purchase:approve` 权限 |
| `submitted` | `draft` | 审批人点击 [驳回] | 退回草稿; 填写驳回原因 |
| `submitted` | `cancelled` | 点击 [取消] | 需确认弹窗 |
| `approved` | `receiving` | 首次执行收货验收 | 至少1项产品已收货 |
| `approved` | `completed` | 一次性全部收货完成 | 所有产品 received_quantity = quantity |
| `receiving` | `completed` | 全部收货完成 | 所有产品 received_quantity = quantity |

### 付款状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `unpaid` | `partial` | 部分付款 | 0 < paid_amount < total_amount |
| `unpaid` | `paid` | 全额付款 | paid_amount = total_amount |
| `partial` | `paid` | 补齐余额 | paid_amount = total_amount |

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求采购单列表 (默认: 按下单日期降序) |
| 2 | 并行请求: 采购单列表 + 汇总统计 + 供应商下拉 + 仓库下拉 + 币种配置 |

| 4 | 统计卡片使用数字滚动动画加载 |
| 5 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建采购单

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建采购单] → 打开新建抽屉 (从右侧滑入, 800px) |
| 2 | 搜索选择供应商 → 自动预填: 币种 (供应商默认币种), 预计到货 (order_date + lead_time_days) |
| 3 | 选择目标仓库 |
| 4 | 点击 [+ 添加产品] → 搜索选择产品; 可查看历史采购价参考 |
| 5 | 填写数量、单价; 小计自动计算 |
| 6 | 填写运费和税率; 税费和总计自动计算 |
| 7 | 底部显示费用汇总 (产品小计 + 运费 + 税费 = 总计) |
| 8 | 点击 [保存草稿] → 保存; 点击 [提交审批] → 保存并提交 |

### INT-04: 供应商联动

| 步骤 | 行为 |
|------|------|
| 1 | 选择供应商后自动填入默认币种 |
| 2 | 自动计算预计到货日期 = 下单日期 + lead_time_days |
| 3 | 如供应商有最低起订金额 → 提交时校验总金额 ≥ min_order_amount |
| 4 | 切换供应商时提示 "更换供应商将重置币种和到货日期，确认继续？" |

### INT-05: 审批操作

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [审批] → 打开审批弹窗 |
| 2 | 弹窗显示采购单摘要 (供应商/仓库/产品明细/费用汇总) |
| 3 | 审批人选择 [通过] 或 [驳回] |
| 4 | 通过 → 状态变为 approved; Toast "审批通过" |
| 5 | 驳回 → 填写驳回原因 (必填); 状态退回 draft; Toast "已驳回" |

### INT-06: 收货验收

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [收货验收] → 打开收货验收弹窗 |
| 2 | 显示所有产品: 采购数量、已收数量、本次收货数量 |
| 3 | 填写本次收货数量 (不超过剩余待收数量) |
| 4 | 填写批次号、有效期 |
| 5 | 选择验收结果: 合格/不合格/部分合格 |
| 6 | 不合格/部分合格 → 必填验收备注 |
| 7 | 确认 → 调用收货 API; 状态变为 receiving 或 completed |
| 8 | 合格产品自动生成入库单 (关联采购单) |
| 9 | 成功 → Toast "收货验收完成，{N} 件产品已入库" |
| 10 | 支持多次收货 (分批到货场景) |

### INT-07: 记录付款

| 步骤 | 行为 |
|------|------|
| 1 | 在详情抽屉点击 [记录付款] → 打开付款弹窗 |
| 2 | 显示总金额、已付金额、待付金额 |
| 3 | 填写本次付款金额、方式、日期 |
| 4 | 确认 → 调用付款 API; 更新 payment_status |
| 5 | 成功 → Toast "付款记录已保存"; 付款状态实时更新 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果 |
| 3 | 生成 Excel 文件 (.xlsx)，包含订单信息 + 产品明细展开 |
| 4 | 文件名格式: `采购管理_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取采购单列表 | GET | `/api/v1/purchase-orders` | `keyword`, `supplier_id`, `warehouse_id`, `status`, `payment_status`, `currency`, `date_start`, `date_end`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取采购单列表 |
| 获取采购汇总 | GET | `/api/v1/purchase-orders/summary` | 同列表筛选参数 (不含分页) | 统计卡片数据 |
| 获取采购单详情 | GET | `/api/v1/purchase-orders/:id` | - | 含产品明细、收货记录、付款记录、操作日志 |
| 创建采购单 | POST | `/api/v1/purchase-orders` | `supplier_id`, `warehouse_id`, `order_date`, `expected_delivery_date`, `currency`, `shipping_cost`, `tax_rate`, `notes`, `items[]` | 新建采购单 (草稿) |
| 更新采购单 | PUT | `/api/v1/purchase-orders/:id` | 同创建参数 | 编辑草稿采购单 |
| 提交采购单 | PATCH | `/api/v1/purchase-orders/:id/submit` | - | 草稿 → 已提交 |
| 审批采购单 | PATCH | `/api/v1/purchase-orders/:id/approve` | `action` (approve/reject), `reason` | 审批通过或驳回 |
| 收货验收 | POST | `/api/v1/purchase-orders/:id/receive` | `items[]` (含 product_id, received_quantity, batch_number, expiry_date, inspection_result, inspection_notes) | 收货并验收; 自动创建入库单 |
| 取消采购单 | PATCH | `/api/v1/purchase-orders/:id/cancel` | `reason` | 取消采购单 |
| 记录付款 | POST | `/api/v1/purchase-orders/:id/payments` | `amount`, `payment_method`, `payment_date`, `transaction_ref`, `notes` | 记录付款 |
| 获取付款记录 | GET | `/api/v1/purchase-orders/:id/payments` | - | 采购单关联的所有付款记录 |
| 批量提交 | POST | `/api/v1/purchase-orders/batch-submit` | `ids[]` | 批量提交草稿 |
| 批量删除 | DELETE | `/api/v1/purchase-orders/batch` | `ids[]` | 批量删除草稿 |
| 导出采购单 | POST | `/api/v1/purchase-orders/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel |
| 获取供应商列表 | GET | `/api/v1/suppliers` | `status=active`, `keyword` | 供应商下拉选项 |
| 获取产品列表 | GET | `/api/v1/products` | `keyword`, `category`, `status=active` | 产品搜索 |
| 获取产品采购历史价 | GET | `/api/v1/products/:id/purchase-history` | `supplier_id` | 历史采购价参考 |
| 获取币种配置 | GET | `/api/v1/settings/currencies` | - | 支持的币种列表和汇率 |
