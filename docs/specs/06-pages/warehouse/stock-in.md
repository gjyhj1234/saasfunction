## 状态机/流转

```
┌──────────┐
│  draft   │ (草稿，可编辑/删除)
└────┬─────┘
     │ 提交
     ▼
┌──────────┐
│submitted │ (已提交，等待审批)
└────┬─────┘
     │ 审批通过              审批驳回
     ▼                      ▼
┌──────────┐          ┌──────────┐
│ approved │          │  draft   │ (退回草稿)
└────┬─────┘          └──────────┘
     │ 确认入库
     ▼
┌──────────┐
│completed │ (已完成，库存已更新)
└──────────┘

--- 取消分支 ---
draft ──────→ cancelled
submitted ──→ cancelled
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `draft` | `submitted` | 点击 [提交] | 验证必填项完整 |
| `draft` | `cancelled` | 点击 [取消] | 需确认弹窗 |
| `submitted` | `approved` | 审批人点击 [通过] | 需 `stock_in:approve` 权限 |
| `submitted` | `draft` | 审批人点击 [驳回] | 退回草稿状态; 填写驳回原因 |
| `submitted` | `cancelled` | 点击 [取消] | 需确认弹窗 |
| `approved` | `completed` | 点击 [确认入库] | 执行库存新增; 生成库存变更记录; 不可撤销 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求入库单列表 (默认: 按创建时间降序) |
| 2 | 并行请求: 入库单列表 + 仓库下拉 + 入库类型枚举 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建入库单

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建入库单] → 打开新建抽屉 (从右侧滑入, 800px) |
| 2 | 选择入库类型 → 联动字段显隐: 采购入库显示关联采购单; 退货入库显示关联退货单; 调拨入库显示来源仓库 |
| 3 | 选择目标仓库 |
| 4 | 点击 [+ 添加产品] → 搜索选择产品; 自动填入产品信息 |
| 5 | 填写数量、批次号、有效期、单价; 小计自动计算 |
| 6 | 可添加多行; 每行可独立删除 |
| 7 | 底部显示合计金额 (实时更新) |
| 8 | 如选择了关联采购单 → 自动导入采购单中的产品明细 |

### INT-04: 提交入库单

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [提交] → 表单验证 (至少1个产品明细) |
| 2 | 弹出确认弹窗 "确认提交？提交后不可修改" |
| 3 | 确认 → 调用提交 API; 状态变为 submitted |
| 4 | 成功 → Toast "入库单已提交"; 刷新列表 |
| 5 | 失败 → Toast 错误提示 |

### INT-05: 审批操作

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [审批] → 打开审批弹窗 |
| 2 | 弹窗显示入库单摘要信息 (类型/仓库/产品明细/金额) |
| 3 | 审批人选择 [通过] 或 [驳回] |
| 4 | 通过 → 状态变为 approved; Toast "审批通过" |
| 5 | 驳回 → 填写驳回原因 (必填); 状态退回 draft; Toast "已驳回" |

### INT-06: 确认入库

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [确认入库] → 确认弹窗 "库存将增加，确认执行入库？" |
| 2 | 确认 → 调用确认入库 API |
| 3 | 后端执行: 逐项增加库存数量; 写入库存变更日志; 更新库位占用 |
| 4 | 成功 → 状态变为 completed; Toast "入库完成，库存已更新" |
| 5 | 失败 → Toast 错误提示; 状态不变 |

### INT-07: 排序交互

| 步骤 | 行为 |
|------|------|
| 1 | 点击可排序列头切换排序: 无 → 升序 → 降序 → 无 |
| 2 | 排序图标: ⬆升序 / ⬇降序 / ⬍默认 |
| 3 | 仅支持单列排序 |
| 4 | 切换排序时保持当前页码 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果 |
| 3 | 生成 Excel 文件 (.xlsx)，包含表格列 + 产品明细展开 |
| 4 | 文件名格式: `入库管理_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取入库单列表 | GET | `/api/v1/stock-movements?type=in` | `keyword`, `sub_type`, `warehouse_id`, `status`, `operator_id`, `date_start`, `date_end`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取入库单列表 |
| 获取入库单详情 | GET | `/api/v1/stock-movements/:id` | - | 含产品明细、操作日志 |
| 创建入库单 | POST | `/api/v1/stock-movements` | `type=in`, `sub_type`, `warehouse_id`, `source_warehouse_id`, `reference_type`, `reference_id`, `cost_currency`, `notes`, `items[]` | 新建入库单 (草稿) |
| 更新入库单 | PUT | `/api/v1/stock-movements/:id` | 同创建参数 | 编辑草稿入库单 |
| 提交入库单 | PATCH | `/api/v1/stock-movements/:id/submit` | - | 草稿 → 已提交 |
| 审批入库单 | PATCH | `/api/v1/stock-movements/:id/approve` | `action` (approve/reject), `reason` | 审批通过或驳回 |
| 确认入库 | PATCH | `/api/v1/stock-movements/:id/complete` | - | 执行入库; 更新库存 |
| 取消入库单 | PATCH | `/api/v1/stock-movements/:id/cancel` | `reason` | 取消入库单 |
| 批量提交 | POST | `/api/v1/stock-movements/batch-submit` | `ids[]` | 批量提交草稿 |
| 批量删除 | DELETE | `/api/v1/stock-movements/batch` | `ids[]` | 批量删除草稿 |
| 导出入库单 | POST | `/api/v1/stock-movements/export?type=in` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel |
| 获取产品列表 | GET | `/api/v1/products` | `keyword`, `category`, `status=active` | 产品搜索 (入库明细) |
| 获取采购单列表 | GET | `/api/v1/purchase-orders` | `status=approved`, `keyword` | 关联采购单搜索 |
| 获取仓库列表 | GET | `/api/v1/warehouses` | `status=active` | 仓库下拉选项 |
