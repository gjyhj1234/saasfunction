## 状态机/流转

```
┌──────────────┐
│ in_progress  │ (进行中，正在盘点)
└──────┬───────┘
       │ 完成盘点 (所有项已录入)
       ▼
┌──────────────┐
│  completed   │ (已完成，差异已记录)
└──────────────┘

--- 取消分支 ---
in_progress ──→ cancelled
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `in_progress` | `completed` | 点击 [完成盘点] | 验证所有盘点项已填写实际数量; 差异项需填写原因; 自动生成库存调整建议 |
| `in_progress` | `cancelled` | 点击 [取消] | 需确认; 已录入数据不保留 |

### 库存调整流程

| 步骤 | 说明 |
|------|------|
| 1 | 盘点完成后，系统自动识别差异项 |
| 2 | 差异项生成调整建议: 盘盈 → 建议入库; 盘亏 → 建议出库 |
| 3 | 管理员逐项审核调整建议: 通过 / 拒绝 |
| 4 | 批准的调整自动生成库存变更记录，更新实际库存 |
| 5 | 调整日志记录: 调整前数量 → 调整后数量 + 原因 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求盘点列表 (默认: 按盘点日期降序) |
| 2 | 并行请求: 盘点列表 + 仓库下拉 + 盘点类型枚举 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建盘点

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建盘点] → 打开新建盘点弹窗 |
| 2 | 选择仓库 → 校验该仓库是否有进行中的盘点; 如有则提示 "该仓库已有进行中的盘点 (IC{编号})，请先完成或取消" |
| 3 | 选择盘点类型: 全盘自动加载全部产品; 部分盘点需选择库区; 抽盘需设置比例 |
| 4 | 选择盘点方式 |
| 5 | 点击 [创建] → 调用创建 API |
| 6 | 成功 → 自动跳转到盘点详情页开始录入; Toast "盘点单已创建" |
| 7 | 系统根据盘点类型自动生成盘点明细列表 (每个产品+库位+批次为一行) |

### INT-04: 录入实际数量

| 步骤 | 行为 |
|------|------|
| 1 | 进入盘点详情 → 显示所有待盘点产品列表 |
| 2 | 手工盘点: 逐行在 "实际数量" 列输入清点数量 |
| 3 | 扫码盘点: 使用条码扫描器扫描产品条码; 自动定位到对应行并填入/累加数量 |
| 4 | 系统自动计算差异 = 实际数量 - 系统数量 |
| 5 | 差异>0 标记绿色 (盘盈); 差异<0 标记红色 (盘亏); 差异=0 无标记 |
| 6 | 差异≠0 时自动展开差异原因下拉框 (必填) |
| 7 | 实时保存已录入数据 (防止页面刷新丢失) |

### INT-05: 差异筛选

| 步骤 | 行为 |
|------|------|
| 1 | 详情页提供 [仅显示差异] 筛选开关 |
| 2 | 开启后仅显示 difference≠0 的行 |
| 3 | 可组合搜索产品名/编码 |
| 4 | 底部汇总实时更新: 盘盈数/盘亏数/总差异 |

### INT-06: 完成盘点

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [完成盘点] → 校验所有项是否已填写实际数量 |
| 2 | 如有未填写项 → Toast "还有 {N} 项未完成盘点" + 高亮未填写行 |
| 3 | 如差异项未填原因 → Toast "请为所有差异项填写差异原因" |
| 4 | 校验通过 → 弹出确认弹窗显示盘点汇总: 总品项/无差异/盘盈/盘亏 |
| 5 | 确认 → 调用完成盘点 API; 状态变为 completed |
| 6 | 成功 → Toast "盘点已完成"; 如有差异则提示 "共 {N} 项差异，已生成库存调整建议" |

### INT-07: 库存调整审核

| 步骤 | 行为 |
|------|------|
| 1 | 盘点完成后，差异项自动生成调整建议列表 |
| 2 | 管理员在详情页查看每项调整建议 |
| 3 | 逐项点击 [批准] 或 [拒绝] |
| 4 | 批准 → 调用库存调整 API; 实际库存更新; 状态变为 "已调整" |
| 5 | 拒绝 → 填写拒绝原因; 状态变为 "已拒绝"; 库存不变 |
| 6 | 支持 [全部批准] 批量操作 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 列表页: 导出盘点记录汇总 |
| 3 | 详情页: 导出完整盘点明细 (含差异分析) |
| 4 | 文件名格式: `库存盘点_{盘点编号}_{YYYYMMDD}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

|------|------|------|----------|------|
| 获取盘点列表 | GET | `/api/v1/inventory-checks` | `keyword`, `check_type`, `check_method`, `warehouse_id`, `status`, `date_start`, `date_end`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取盘点列表 |
| 获取盘点详情 | GET | `/api/v1/inventory-checks/:id` | - | 含盘点信息 + 汇总统计 |
| 获取盘点明细 | GET | `/api/v1/inventory-checks/:id/items` | `keyword`, `has_difference`, `page`, `page_size` | 分页获取盘点明细; has_difference=true 仅返回差异项 |
| 创建盘点 | POST | `/api/v1/inventory-checks` | `warehouse_id`, `check_type`, `check_method`, `check_date`, `scope_zones[]`, `spot_rate`, `notes` | 新建盘点; 自动生成明细 |
| 更新盘点明细 | PATCH | `/api/v1/inventory-checks/:id/items/:item_id` | `actual_quantity`, `difference_reason`, `difference_notes` | 录入/更新单项实际数量 |
| 批量更新明细 | PATCH | `/api/v1/inventory-checks/:id/items/batch` | `items[]` (含 item_id, actual_quantity, difference_reason) | 批量更新多项 |
| 完成盘点 | PATCH | `/api/v1/inventory-checks/:id/complete` | - | 完成盘点; 生成调整建议 |
| 取消盘点 | PATCH | `/api/v1/inventory-checks/:id/cancel` | `reason` | 取消盘点 |
| 审核调整 | PATCH | `/api/v1/inventory-checks/:id/items/:item_id/adjust` | `action` (approve/reject), `reason` | 审核单项库存调整 |
| 批量审核调整 | POST | `/api/v1/inventory-checks/:id/batch-adjust` | `item_ids[]`, `action` (approve/reject) | 批量审核调整 |
| 导出盘点 | POST | `/api/v1/inventory-checks/:id/export` | - | 导出盘点详情 Excel |
| 导出盘点列表 | POST | `/api/v1/inventory-checks/export` | 同列表筛选参数 | 导出列表汇总 Excel |
