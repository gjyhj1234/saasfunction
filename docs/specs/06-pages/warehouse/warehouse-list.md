## 页面信息

|------|-----|

## 状态机/流转

```
┌──────────┐
│  active  │ (启用，正常使用中)
└────┬─────┘
     │ 手动停用
     ▼
┌──────────┐
│ disabled │ (停用，无法出入库)
└────┬─────┘
     │ 手动启用
     ▼
┌──────────┐
│  active  │ (重新启用)
└──────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `active` | `disabled` | 点击 [停用] 按钮 | 需确认; 如仓库有库存则提示 "该仓库当前有库存，请先完成库存转移后再停用" |
| `disabled` | `active` | 点击 [启用] 按钮 | 无需确认; 直接启用 |

### 库位状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `active` | `maintenance` | 手动标记维护 | 维护期间不接受新入库 |
| `active` | `disabled` | 手动停用 | 库位停用; 需先清空库存 |
| `maintenance` | `active` | 维护完成 | 恢复正常使用 |
| `disabled` | `active` | 手动启用 | 重新启用库位 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求仓库列表 (默认筛选: status=active) |
| 2 | 并行请求: 仓库列表 + 仓库类型枚举 + 管理员列表 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建仓库

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建仓库] → 打开新建仓库抽屉 (从右侧滑入, 640px) |
| 2 | 抽屉标题: "新建仓库" |
| 3 | 填写基本信息; 仓库编码输入时实时查重 |
| 4 | 选择仓库类型; 如选 main 则校验是否已有主仓库 |
| 5 | 选择管理员 (搜索选择) |
| 6 | 点击 [保存] → 表单验证 → 调用创建 API |
| 7 | 成功 → Toast "仓库创建成功"; 关闭抽屉; 刷新列表 |
| 8 | 失败 → Toast 错误提示; 抽屉保持打开 |

### INT-04: 编辑仓库

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [编辑] → 打开编辑仓库抽屉，预填当前数据 |
| 2 | 仓库编码字段只读不可修改 |
| 3 | 其余字段可正常编辑 |
| 4 | 点击 [保存] → 调用更新 API |
| 5 | 成功 → Toast "保存成功"; 关闭抽屉; 刷新列表 |

### INT-05: 查看详情抽屉

| 步骤 | 行为 |
|------|------|
| 1 | 点击仓库编码 / [查看] → 打开详情抽屉 (640px) |
| 2 | 上半部分显示仓库基本信息 |
| 3 | 下半部分显示库位列表 (子表格，分页每页 10 条) |
| 4 | 库位列表支持搜索 (按库位编码/名称) |
| 5 | 可在详情抽屉内直接 [新增库位] / [编辑库位] / [删除库位] |

### INT-06: 库位管理

| 步骤 | 行为 |
|------|------|
| 1 | 在详情抽屉点击 [+ 新增库位] → 展开内嵌表单 |
| 2 | 填写库位信息; 编码同仓库内唯一校验 |
| 3 | 温度区域仅在仓库 temperature_controlled=true 时可选 |
| 4 | 点击 [保存] → 调用创建库位 API |
| 5 | 成功 → 刷新库位列表; Toast "库位创建成功" |
| 6 | 编辑/删除库位同理，删除需确认弹窗 |

### INT-07: 停用仓库

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [停用] → 弹出确认弹窗 |
| 2 | 校验仓库是否有库存 (total_stock > 0) |
| 3 | 如有库存 → 弹窗显示警告 "该仓库当前有 {N} 种产品共 {M} 件库存，请先完成库存转移" + [取消] 按钮 |
| 4 | 如无库存 → 确认后调用停用 API |
| 5 | 成功 → Toast "仓库已停用"; 行内状态实时更新 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果全部数据 |
| 3 | 生成 Excel 文件 (.xlsx)，包含所有表格列 |
| 4 | 文件名格式: `仓库列表_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

## 空状态

| 场景 | 展示内容 |
|------|----------|
| 无仓库记录 | 仓库插画 + "暂无仓库信息" + [新建仓库] 按钮 |
| 搜索无结果 | 搜索插画 + "未找到匹配的仓库" + "请尝试其他搜索词或调整筛选条件" |
| 筛选无结果 | 筛选插画 + "没有符合条件的仓库记录" + [清除筛选] 链接按钮 |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取仓库列表 | GET | `/api/v1/warehouses` | `keyword`, `type`, `temperature_controlled`, `status`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取仓库列表 |
| 获取仓库详情 | GET | `/api/v1/warehouses/:id` | - | 含库存汇总信息 |
| 创建仓库 | POST | `/api/v1/warehouses` | `name`, `code`, `type`, `address`, `area_sqm`, `temperature_controlled`, `manager_id`, `status` | 新建仓库 |
| 更新仓库 | PUT | `/api/v1/warehouses/:id` | 同创建参数 (code 不可修改) | 编辑仓库信息 |
| 停用仓库 | PATCH | `/api/v1/warehouses/:id/disable` | - | 停用仓库 |
| 启用仓库 | PATCH | `/api/v1/warehouses/:id/enable` | - | 启用仓库 |
| 获取库位列表 | GET | `/api/v1/warehouses/:id/locations` | `keyword`, `zone`, `status`, `page`, `page_size` | 获取仓库下的库位分页列表 |
| 创建库位 | POST | `/api/v1/warehouses/:id/locations` | `name`, `code`, `zone`, `shelf`, `layer`, `max_weight_kg`, `temperature_zone`, `status` | 新增库位 |
| 更新库位 | PUT | `/api/v1/warehouses/:id/locations/:loc_id` | 同创建参数 | 编辑库位信息 |
| 删除库位 | DELETE | `/api/v1/warehouses/:id/locations/:loc_id` | - | 删除空库位; 有库存则拒绝 |
| 导出仓库 | POST | `/api/v1/warehouses/export` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel 文件 |
| 获取管理员列表 | GET | `/api/v1/users` | `role=warehouse_manager`, `status=active` | 可选管理员列表 |
