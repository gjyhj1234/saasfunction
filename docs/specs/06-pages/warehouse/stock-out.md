## 状态机/流转

```
┌──────────┐
│  draft   │ (草稿，可编辑/删除)
└────┬─────┘
     │ 提交
     ▼
┌──────────┐
│submitted │ (已提交，等待审批)
└────┬─────┘
     │ 审批通过              审批驳回
     ▼                      ▼
┌──────────┐          ┌──────────┐
│ approved │          │  draft   │ (退回草稿)
└────┬─────┘          └──────────┘
     │ 确认出库
     ▼
┌──────────┐
│completed │ (已完成，库存已扣减)
└──────────┘

--- 取消分支 ---
draft ──────→ cancelled
submitted ──→ cancelled
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 | 说明 |
|----------|----------|----------|------|
| `draft` | `submitted` | 点击 [提交] | 验证必填项完整; 验证库存充足 |
| `draft` | `cancelled` | 点击 [取消] | 需确认弹窗 |
| `submitted` | `approved` | 审批人点击 [通过] | 需 `stock_out:approve` 权限; 再次校验库存 |
| `submitted` | `draft` | 审批人点击 [驳回] | 退回草稿状态; 填写驳回原因 |
| `submitted` | `cancelled` | 点击 [取消] | 需确认弹窗 |
| `approved` | `completed` | 点击 [确认出库] | 执行库存扣减; 实时校验库存是否充足; 不可撤销 |

## 交互说明

### INT-01: 页面加载

| 步骤 | 行为 |
|------|------|
| 1 | 页面加载时请求出库单列表 (默认: 按创建时间降序) |
| 2 | 并行请求: 出库单列表 + 仓库下拉 + 科室列表 + 出库类型枚举 |
| 3 | 表格使用骨架屏 (Skeleton) 作为加载占位 |
| 4 | 检查 URL query 参数，如有筛选条件则使用 URL 中的条件 |

### INT-02: 搜索交互

| 步骤 | 行为 |
|------|------|
| 1 | 输入关键词，300ms 防抖后自动搜索 |
| 2 | 搜索时表格显示 loading 状态 |
| 3 | 搜索结果高亮匹配文字 |
| 4 | 清空搜索框 → 恢复原筛选结果 |

### INT-03: 新建出库单

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [新建出库单] → 打开新建抽屉 (从右侧滑入, 800px) |
| 2 | 选择出库类型 → 联动字段显隐: 销售出库显示接收方; 领用出库显示科室/医生; 调拨出库显示目标仓库 |
| 3 | 选择来源仓库 (切换仓库时清空已选产品并提示) |
| 4 | 点击 [+ 添加产品] → 搜索产品 (仅显示所选仓库有库存的产品) |
| 5 | 选择产品后自动加载可用批次列表 (按 FIFO 排序; 最早批次置顶并推荐) |
| 6 | 填写出库数量; 实时校验: 如超出可用库存 → 输入框红色边框 + 提示 "库存不足，当前可用: {N}" |
| 7 | 底部显示合计金额 (实时更新) |
| 8 | 可添加多行; 同一产品不同批次可分多行 |

### INT-04: FIFO 批次推荐

| 步骤 | 行为 |
|------|------|
| 1 | 选择产品后，系统加载该产品在所选仓库的所有可用批次 |
| 2 | 批次按入库时间升序排列 (先入先出) |
| 3 | 默认自动选中最早批次 |
| 4 | 批次下拉显示: 批次号 + 入库日期 + 有效期 + 可用数量 |
| 5 | 即将过期的批次 (≤90天) 标记 ⚠️ 黄色警告 |
| 6 | 已过期批次标记 🔴 且不可选择 |

### INT-05: 库存不足校验

| 步骤 | 行为 |
|------|------|
| 1 | 填写出库数量时实时校验可用库存 |
| 2 | 如数量 > 可用库存 → 输入框红色边框 + 错误提示 |
| 3 | 提交时再次全量校验所有行的库存 |
| 4 | 如有库存不足行 → 阻止提交; 高亮问题行; Toast "以下产品库存不足" |
| 5 | 审批和确认出库时也进行实时库存校验 |

### INT-06: 提交出库单

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [提交] → 表单验证 (至少1个产品明细; 所有数量校验通过) |
| 2 | 弹出确认弹窗 "确认提交？提交后不可修改" |
| 3 | 确认 → 调用提交 API; 状态变为 submitted |
| 4 | 成功 → Toast "出库单已提交"; 刷新列表 |
| 5 | 失败 (如库存变更导致不足) → Toast 错误提示 "库存已变更，请重新确认数量" |

### INT-07: 确认出库

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [确认出库] → 确认弹窗 "库存将扣减，确认执行出库？" |
| 2 | 确认 → 调用确认出库 API |
| 3 | 后端执行: 逐项扣减库存; 写入库存变更日志; 更新库位占用; FIFO 扣减 |
| 4 | 成功 → 状态变为 completed; Toast "出库完成，库存已更新" |
| 5 | 失败 (库存不足) → Toast "出库失败: {产品名} 库存不足"; 状态不变 |

### INT-08: 导出功能

| 步骤 | 行为 |
|------|------|
| 1 | 点击 [导出] 按钮 |
| 2 | 如有选中行 → 导出选中行; 无选中行 → 导出当前筛选结果 |
| 3 | 生成 Excel 文件 (.xlsx)，包含表格列 + 产品明细展开 |
| 4 | 文件名格式: `出库管理_{YYYYMMDD_HHmmss}.xlsx` |
| 5 | 导出完成 → Toast 提示 "导出成功" |

## API 数据源

| 接口 | 方法 | 路径 | 请求参数 | 说明 |
|------|------|------|----------|------|
| 获取出库单列表 | GET | `/api/v1/stock-movements?type=out` | `keyword`, `sub_type`, `warehouse_id`, `status`, `department_id`, `operator_id`, `date_start`, `date_end`, `sort_field`, `sort_order`, `page`, `page_size` | 分页获取出库单列表 |
| 获取出库单详情 | GET | `/api/v1/stock-movements/:id` | - | 含产品明细、操作日志 |
| 创建出库单 | POST | `/api/v1/stock-movements` | `type=out`, `sub_type`, `warehouse_id`, `target_warehouse_id`, `department_id`, `doctor_id`, `receiver`, `cost_currency`, `notes`, `items[]` | 新建出库单 (草稿) |
| 更新出库单 | PUT | `/api/v1/stock-movements/:id` | 同创建参数 | 编辑草稿出库单 |
| 提交出库单 | PATCH | `/api/v1/stock-movements/:id/submit` | - | 草稿 → 已提交; 校验库存 |
| 审批出库单 | PATCH | `/api/v1/stock-movements/:id/approve` | `action` (approve/reject), `reason` | 审批通过或驳回 |
| 确认出库 | PATCH | `/api/v1/stock-movements/:id/complete` | - | 执行出库; 扣减库存 |
| 取消出库单 | PATCH | `/api/v1/stock-movements/:id/cancel` | `reason` | 取消出库单 |
| 批量提交 | POST | `/api/v1/stock-movements/batch-submit` | `ids[]` | 批量提交草稿 |
| 批量删除 | DELETE | `/api/v1/stock-movements/batch` | `ids[]` | 批量删除草稿 |
| 导出出库单 | POST | `/api/v1/stock-movements/export?type=out` | 同列表筛选参数 + `ids[]` (可选) | 导出 Excel |
| 获取产品库存 | GET | `/api/v1/warehouses/:id/stock` | `product_id`, `keyword` | 获取指定仓库中产品的可用库存和批次 |
| 获取批次列表 | GET | `/api/v1/warehouses/:id/stock/:product_id/batches` | - | 获取产品可用批次 (FIFO 排序) |
| 获取科室列表 | GET | `/api/v1/departments` | `status=active` | 科室下拉选项 |
| 获取仓库列表 | GET | `/api/v1/warehouses` | `status=active` | 仓库下拉选项 |
