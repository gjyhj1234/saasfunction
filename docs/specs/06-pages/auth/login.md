## 交互说明

### INT-01: 表单验证

| 步骤 | 行为 |
|------|------|
| 1 | 用户名/密码输入框失焦时触发单字段验证 |
| 2 | 验证不通过: 输入框边框变红，下方显示红色错误提示文字 |
| 3 | 错误提示文案: 用户名 — "请输入用户名" / "用户名至少 3 个字符"；密码 — "请输入密码" / "密码至少 6 个字符" |
| 4 | 重新输入时，错误提示实时消失 |

### INT-02: 登录提交

| 步骤 | 行为 |
|------|------|
| 1 | 用户点击 [登录] 按钮或按 Enter 键触发提交 |
| 2 | 前端先进行全量表单验证，不通过则聚焦到第一个错误字段 |
| 3 | 验证通过后，[登录] 按钮进入 loading 状态（显示 Spinner + "登录中..."），禁止重复点击 |
| 4 | 调用 `POST /api/auth/login` 接口 |
| 5-成功 | 存储 token（accessToken + refreshToken），跳转至 `/dashboard` |
| 5-失败 | 按钮恢复可点击状态，根据错误码显示 Toast 提示 |

### INT-03: 错误处理

| 错误码 | Toast 提示 | 附加行为 |
|--------|-----------|----------|
| 401 (用户名或密码错误) | "用户名或密码错误，请重新输入" | 清空密码输入框，聚焦到密码字段 |
| 423 (账户已锁定) | "账户已锁定，请 {remainingMinutes} 分钟后重试" | 禁用登录按钮，显示倒计时 |
| 403 (账户已停用) | "该账户已停用，请联系管理员" | — |
| 429 (请求过于频繁) | "操作过于频繁，请稍后再试" | — |
| 500 (服务器错误) | "服务异常，请稍后重试" | — |
| 网络错误 | "网络连接失败，请检查网络设置" | — |

### INT-04: 登录失败锁定机制

| 属性 | 规范 |
|------|------|
| 锁定阈值 | 连续 5 次登录失败 |
| 锁定时长 | 30 分钟 |
| 计数重置 | 成功登录后重置失败计数 |
| 锁定提示 | 第 3 次失败起，Toast 提示 "您还有 {remaining} 次尝试机会" |
| 锁定后行为 | 登录按钮禁用，显示灰色，文字变为 "账户已锁定 (MM:SS)" 倒计时 |
| 服务端校验 | 锁定状态由服务端维护，防止前端绕过 |

### INT-05: 记住我

| 属性 | 规范 |
|------|------|
| 未勾选 | accessToken 有效期 24 小时，refreshToken 有效期 7 天 |
| 已勾选 | accessToken 有效期 7 天，refreshToken 有效期 30 天 |
| 用户名记忆 | 勾选后将用户名存储到 localStorage，下次打开自动填充 |

### INT-06: 忘记密码

| 步骤 | 行为 |
|------|------|
| 1 | 点击 "忘记密码？" 链接 |
| 2 | 跳转至 `/forgot-password` 页面（或弹出重置密码弹窗） |
| 3 | 用户输入注册邮箱，系统发送重置密码邮件 |
| 4 | 邮件内含重置链接，有效期 30 分钟 |

### INT-07: 已登录用户访问

| 规则 | 说明 |
|------|------|
| token 有效 | 访问 `/login` 自动重定向到 `/dashboard` |
| token 过期 | 尝试使用 refreshToken 静默刷新；失败则停留在登录页 |

### INT-08: 键盘交互

| 按键 | 行为 |
|------|------|
| Tab | 在用户名 → 密码 → 记住我 → 登录按钮之间切换焦点 |
| Enter | 在任意输入框按 Enter 触发登录 |
| Esc | 清空当前输入框 |

## API 数据源

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 登录 | POST | `/api/auth/login` | 请求体: `{ username, password, rememberMe }` |
| 刷新Token | POST | `/api/auth/refresh` | 请求体: `{ refreshToken }` |
| 忘记密码 | POST | `/api/auth/forgot-password` | 请求体: `{ email }` |

### 登录接口请求/响应

**请求体:**
```json
{
  "username": "string",
  "password": "string",
  "rememberMe": false
}
```

**成功响应 (200):**
```json
{
  "code": 200,
  "data": {
    "accessToken": "string",
    "refreshToken": "string",
    "expiresIn": 86400,
    "user": {
      "id": "string",
      "username": "string",
      "name": "string",
      "role": "string",
      "tenantId": "string",
      "clinicId": "string",
      "language": "en",
      "permissions": ["patient:read", "patient:create", "..."]
    }
  }
}
```

**失败响应:**
```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": {
    "remainingAttempts": 3
  }
}
```

## 安全规范

| 属性 | 规范 |
|------|------|
| 密码传输 | 全程 HTTPS，密码不做前端加密（依赖 TLS） |
| Token 存储 | accessToken 存储在内存 (变量)；refreshToken 存储在 httpOnly cookie |
| XSS 防护 | 输入框禁止 HTML 标签注入 |
| CSRF 防护 | 使用 SameSite cookie + CSRF token |
| 暴力破解防护 | 服务端限流 + 账户锁定机制 |
