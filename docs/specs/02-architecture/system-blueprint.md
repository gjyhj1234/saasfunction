## 服务划分

| 服务 | 职责 | 技术栈 |
|------|------|--------|
| api-gateway | 请求路由、认证、限流、多语言 | NestJS |
| appointment-service | 预约管理 | NestJS + TypeORM |
| patient-service | 患者档案 | NestJS + TypeORM |
| billing-service | 收费结算、多币种 | NestJS + TypeORM |
| medical-record-service | 电子病历 | NestJS + TypeORM |
| warehouse-service | 仓储管理(WMS): 仓库/库位/进销存/采购 | NestJS + TypeORM |
| mall-service | 商城系统: 商品/订单/支付/物流 | NestJS + TypeORM |
| inventory-service | 临床库存: 诊室级耗材领用 | NestJS + TypeORM |
| report-service | 报表分析 | NestJS + TypeORM |
| auth-service | 用户认证与权限 | NestJS + Passport |
| notification-service | 消息通知(WhatsApp/LINE/SMS) | NestJS |
| i18n-service | 国际化与本地化配置 | NestJS |

## 部署架构

- **容器编排**: Kubernetes
- **CI/CD**: GitHub Actions
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack
- **CDN**: CloudFlare
- **地域**: 东南亚节点优先 (AWS ap-southeast / GCP asia-southeast)

## 通信模式

- 同步: REST API (服务间)
- 异步: 消息队列 (事件驱动，尤其用于仓储↔商城库存同步)
- 实时: WebSocket (通知推送)
