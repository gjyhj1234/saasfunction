# 验收测试文档

> 生成时间: 2026-04-15T09:28:33Z
> 执行范围: mvp
> 目标市场: 东南亚 (Southeast Asia)

## 测试策略

### 测试层次
1. **单元测试**: 覆盖核心业务逻辑 (≥80%)
2. **集成测试**: API 端到端验证
3. **E2E 测试**: 关键业务流程
4. **性能测试**: 并发与响应时间

## 验收测试用例 — 诊所管理

### AT-001: 用户登录
| 项目 | 内容 |
|------|------|
| 前提 | 用户账号已创建 |
| 步骤 | 1. 打开登录页 2. 输入账号密码 3. 点击登录 |
| 预期 | 跳转到工作台，显示用户信息 |
| 异常 | 密码错误时显示提示，5次锁定 |

### AT-002: 创建预约
| 项目 | 内容 |
|------|------|
| 前提 | 用户已登录，有可用医生和时段 |
| 步骤 | 1. 进入预约管理 2. 点击新建 3. 选择患者 4. 选择医生 5. 选择时段 6. 确认 |
| 预期 | 预约创建成功，显示在日历中 |
| 异常 | 时间冲突时给出提示 |

### AT-003: 预约改期
| 项目 | 内容 |
|------|------|
| 前提 | 存在一个 scheduled 状态的预约 |
| 步骤 | 1. 打开预约详情 2. 点击改期 3. 选择新时段 4. 确认 |
| 预期 | 预约时间更新，通知患者 |
| 异常 | 新时段冲突时给出提示 |

### AT-004: 预约取消
| 项目 | 内容 |
|------|------|
| 前提 | 存在一个 scheduled/confirmed 的预约 |
| 步骤 | 1. 打开预约详情 2. 点击取消 3. 填写原因 4. 确认 |
| 预期 | 预约状态变为 cancelled |
| 异常 | 已完成的预约不可取消 |

### AT-005: 新建患者
| 项目 | 内容 |
|------|------|
| 前提 | 用户已登录，有患者管理权限 |
| 步骤 | 1. 进入患者管理 2. 点击新建 3. 填写信息 4. 保存 |
| 预期 | 患者创建成功，出现在列表中 |
| 异常 | 手机号重复时给出提示 |

### AT-006: 患者搜索
| 项目 | 内容 |
|------|------|
| 前提 | 系统中有患者数据 |
| 步骤 | 1. 进入患者列表 2. 输入搜索关键词 3. 回车 |
| 预期 | 列表显示匹配结果 |
| 异常 | 无匹配时显示空状态 |

### AT-007: 创建收费单（多币种）
| 项目 | 内容 |
|------|------|
| 前提 | 存在已完成的诊疗预约 |

| 预期 | 账单以选择的币种创建成功，状态为 pending |
| 异常 | 金额为 0 时不允许提交 |

### AT-008: 收款（东南亚支付方式）
| 项目 | 内容 |
|------|------|
| 前提 | 存在 pending 的收费单 |

|------|------|
| 前提 | 存在多个租户 |

## 验收测试用例 — 商城

|------|------|

| 预期 | 商品在前台商城可见 |

|------|------|

| 异常 | 库存不足时提示 |

### AT-020: 商城库存同步
| 项目 | 内容 |
|------|------|
| 前提 | 商城商品关联仓储库存 |
| 步骤 | 1. 仓储中入库 2. 查看商城可售数量 |
| 预期 | 商城可售数量与仓储库存保持同步 |
| 异常 | 库存变动事件丢失时有补偿机制 |
